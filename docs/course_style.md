# Стиль глав курса

Эта страница — внутренний стандарт курса. Она нужна, чтобы главы были не литературным рассказом, а удобным учебным модулем.

## Цель

Каждая глава должна работать в трёх режимах:

| Режим | Что нужно ученику |
|---|---|
| Первое чтение | понять идею без перегруза |
| Повторение перед задачей | быстро найти шаблон и ошибку |
| Повторение перед экзаменом | освежить правила, схемы и чеклист |

Главная формула главы:

```text
правило -> схема -> шаблон -> пример -> ошибка -> практика
```

Не нужно превращать курс в сухую справку. Но главное правило должно появляться раньше длинного объяснения.

## Правило абзацев

- Один абзац — максимум 3–4 строки Markdown.
- Если абзац длиннее, разбей его или преврати в таблицу.
- Один раздел = одна мысль.
- Длинные мотивационные вступления лучше заменить блоком `За 30 секунд`.

Плохо:

> В C++ ты привык к удобным конструкциям, но процессор смотрит на программу иначе, потому что он исполняет инструкции, и поэтому нам нужно постепенно разобраться...

Лучше:

> Процессор не видит `while`. Он видит метку, проверку и переход.

---

## Правило технического паттерна

Каждый важный паттерн должен иметь 4 слоя:

1. C++-смысл.
2. Low-level shape.
3. NASM-код.
4. Типовая ошибка.

Пример:

```cpp
answer = a + b;
```

```text
load a -> add b -> print answer
```

```asm
mov eax, [a]
add eax, [b]
```

Типовая ошибка:

```asm
mov eax, a      ; это адрес, а не значение
add eax, [b]
```

## Правило практики

В ключевых главах нужна практика четырёх типов.

### A. Trace

Ученик проходит код по строкам и заполняет регистры/память/флаги.

### B. Заполни пропуски

Ученик дописывает 2–5 пропущенных команд или операндов.

### C. Напиши сам

Ученик пишет маленький фрагмент с нуля.

### D. Найди баг

Ученик получает неправильный код и объясняет ошибку.

Так тренируются разные навыки: чтение, восстановление, самостоятельное письмо и отладка.

---

## Что не сокращать слишком сильно

Эти темы должны оставаться подробными:

- `x` vs `[x]`;
- `scanf` / `printf`;
- `cdq` + `idiv`;
- `cmp`, `test`, `jcc`;
- signed vs unsigned jumps;
- стек, `call`, `ret`;
- CDECL и frame layout;
- `lea` vs `mov`;
- x87 и `printf("%f")`.

Их можно делать структурнее, но нельзя превращать в одну строку.

---

## Что считать готовой главой

Глава готова, если:

- есть `За 30 секунд`;
- есть `Минимум после главы`;
- есть главная схема или таблица;
- есть базовый шаблон, если тема практическая;
- есть хотя бы один разбор примера;
- есть типовые ошибки;
- есть практика минимум двух типов;
- финальный чеклист проверяет действие, а не ощущение “я понял”.
