# День 16. Стек: стопка, которая растёт вниз

## Опора на материалы ВШЭ

`Slides2026-07.pdf`: стек IA-32, `push`, `pop`, `call`, `ret`.

## Зачем этот день

Стек нужен для функций, рекурсии, аргументов, адресов возврата и локальных переменных. Без него CDECL и фреймы будут казаться магией.

## Главная мысль

Стек растёт к меньшим адресам. `esp` указывает на верхушку.

## `push`

```asm
push eax
```

примерно значит:

```text
esp = esp - 4
[esp] = eax
```

Картинка:

```text
Before push eax:
0x100C | old data |
0x1008 | old data | <- esp

After push eax:
0x100C | old data |
0x1008 | old data |
0x1004 |   eax    | <- esp
```

## `pop`

```asm
pop eax
```

примерно значит:

```text
eax = [esp]
esp = esp + 4
```

## `call` и `ret`

`call f` — это не просто прыжок. Это прыжок с запиской, куда вернуться.

```text
call f:
1. push address_after_call
2. jump to f

ret:
1. pop return_address
2. jump to return_address
```

## Пример

```asm
push 10
push 20
pop eax     ; eax = 20
pop ebx     ; ebx = 10
```

Стек работает LIFO: last in, first out.
