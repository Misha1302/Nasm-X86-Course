# День 04. Почему `eax`, `ax`, `ah`, `al` — это не четыре переменные

## Опора на материалы ВШЭ

`Slides2026-02.pdf`: регистры IA-32 и размеры операндов.

## Зачем этот день

В C++ переменная обычно отдельная. В x86 имя регистра может означать весь регистр или его кусок. Это важно для байтовых задач, сдвигов и чтения листингов.

## Главная мысль

`eax`, `ax`, `ah`, `al` перекрываются.

```text
31                              16 15       8 7        0
+--------------------------------+----------+----------+
|          high 16 bits          |    AH    |    AL    |
+--------------------------------+----------+----------+
                                  \__________ _________/
                                             |
                                             AX

EAX = 32 bits
AX  = low 16 bits
AH  = high byte of AX
AL  = low byte of AX
```

## Пример руками

```text
eax = 0x12345678
ax  =     0x5678
ah  =       0x56
al  =       0x78
```

Если сделать:

```asm
mov al, 0
```

то `eax` станет:

```text
eax = 0x12345600
```

Поменялся только младший байт.

## Основные регистры, которые встретятся чаще всего

| Register | Роль в курсе |
|---|---|
| `eax` | результат, аккумулятор, return value |
| `ebx` | общий регистр, в функциях часто надо сохранять |
| `ecx` | счётчик, переменный сдвиг через `cl` |
| `edx` | остаток/старшая половина при делении |
| `esi`, `edi` | индексы/указатели |
| `esp` | вершина стека |
| `ebp` | база текущего фрейма функции |
| `eip` | адрес следующей инструкции |
| `eflags` | флаги результата |

## Размеры операндов

| Размер | Биты | Байты | Похоже на C/C++ |
|---|---:|---:|---|
| `byte` | 8 | 1 | `char` |
| `word` | 16 | 2 | `short` |
| `dword` | 32 | 4 | `int`, pointer в IA-32 |
| `qword` | 64 | 8 | `long long`, `double` |

## Мини-челленджи

1. Для `eax = 0xDEADBEEF` найди `ax`, `ah`, `al`.
2. Что будет с `eax = 0x11223344` после `mov al, 0x99`?
3. Почему переменный сдвиг часто использует `cl`, а не весь `ecx`?

<details>
<summary>Ответы / подсказки</summary>

1. `ax = 0xBEEF`, `ah = 0xBE`, `al = 0xEF`.
2. `eax = 0x11223399`.
3. В x86 количество сдвига для таких команд берётся из `cl`, младшего байта `ecx`.

</details>

## Типовая ошибка

Думать, что `eax` и `al` независимы. Нет: `al` — это часть `eax`.
