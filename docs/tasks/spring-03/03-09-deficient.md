# 03-9. Недостаточные числа

## Условие

Недостаточное число — натуральное число, у которого сумма собственных делителей меньше самого числа.

Нужно вывести `K`-е недостаточное число.

Проверку числа нужно вынести в отдельную функцию по `cdecl`.

## Ввод

```text
K
```

## Вывод

Одно число: `K`-е недостаточное число.

## Ограничения

- `K <= 7000`;
- функция проверки должна соблюдать `cdecl`.

## Решение

Основная программа перебирает натуральные числа и считает, сколько из них недостаточные.

```text
count = 0
x = 1
while count < K:
    if is_deficient(x):
        count++
    x++
answer = x - 1
```

## Функция `is_deficient(n)`

Нужно посчитать сумму собственных делителей.

Быстрее идти до `sqrt(n)`:

```text
sum = 0
for d = 1; d*d <= n; d++:
    if n % d == 0:
        pair = n / d

        if d != n:
            sum += d

        if pair != d and pair != n:
            sum += pair

return sum < n
```

Для `n = 1` сумма собственных делителей равна `0`, значит `1` недостаточное.

## NASM-shape

Функция:

```asm
is_deficient:
    push ebp
    mov ebp, esp
    ; [ebp+8] = n
    ; eax = 1 если недостаточное, иначе 0
    mov esp, ebp
    pop ebp
    ret
```

Проверка делимости:

```asm
mov eax, [n]
cdq
idiv dword [d]
; edx = remainder
```

Для натуральных чисел можно использовать `xor edx, edx` + `div`.

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| добавить само число `n` в сумму | собственные делители не включают число |
| дважды добавить квадратный корень | если `d == n/d`, добавлять нужно один раз |
| нарушить `cdecl` | задача явно требует отдельную функцию |
| хранить счётчик в caller-saved регистре через call | `eax/ecx/edx` могут быть испорчены |
