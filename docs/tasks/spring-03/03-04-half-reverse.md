# 03-4. Разворот половины последовательности

## Коротко

Массивы запрещены, значит порядок храним через рекурсивные вызовы.

<details>
<summary>Идея решения</summary>

Нужно вывести:

```text
1-й, 3-й, 5-й, ...
потом чётные позиции в обратном порядке
```

Рекурсивная функция читает очередной элемент и номер позиции.

- нечётную позицию печатаем до рекурсивного вызова;
- чётную позицию сохраняем и печатаем после рекурсивного вызова.

Так стек вызовов сам разворачивает чётные элементы.

</details>

## Где может сломаться

- вывести `0`-терминатор;
- потерять значение после recursive call;
- перепутать чётность позиции;
- забыть очистить стек после вызова функции.
