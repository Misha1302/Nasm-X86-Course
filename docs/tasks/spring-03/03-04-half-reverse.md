# 03-4. Разворот половины последовательности

## Условие

Дана последовательность ненулевых целых чисел, заканчивающаяся нулём. Ноль в последовательность не входит.

Нужно вывести:

1. элементы с нечётными позициями исходной последовательности: 1-й, 3-й, 5-й, ...;
2. затем элементы с чётными позициями в обратном порядке.

Массивы запрещены. Нужно использовать рекурсивную функцию, каждый вызов которой требует `O(1)` памяти.

## Ввод

```text
a1 a2 a3 ... 0
```

## Вывод

Переставленная последовательность.

## Ограничения

- элементы — signed 32-bit;
- массивы использовать нельзя;
- память на один recursive call — `O(1)`.

## Решение

Рекурсия должна читать пару чисел:

```text
oddPositionValue
evenPositionValue
```

Первое число пары печатаем сразу. Второе число пары печатаем после рекурсивного вызова.

Так чётные позиции автоматически выйдут в обратном порядке при размотке стека.

## Алгоритм

```text
function solvePair():
    read x
    if x == 0:
        return

    print x       ; нечётная позиция

    read y
    if y == 0:
        return

    solvePair()

    print y       ; чётная позиция после рекурсии
```

Пример:

```text
1 2 3 4 5 6 0
```

Ход:

```text
print 1
  print 3
    print 5
    after recursion print 6
  after recursion print 4
after recursion print 2
```

Результат:

```text
1 3 5 6 4 2
```

## NASM-shape

Функция — обычная `cdecl`-функция:

```asm
solve_pair:
    push ebp
    mov ebp, esp
    sub esp, 8      ; locals: x, y

    ; read x
    ; if x == 0 -> return
    ; print x

    ; read y
    ; if y == 0 -> return

    call solve_pair

    ; print y

.return:
    mov esp, ebp
    pop ebp
    ret
```

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| сохранить всю последовательность | массивы запрещены |
| печатать чётный элемент до рекурсии | чётные выйдут в прямом порядке |
| не сохранить `y` перед рекурсией | после `call` регистры могут быть испорчены |
| забыть базовый случай на `0` | бесконечное чтение / неверный вывод |
