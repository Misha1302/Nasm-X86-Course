# 01-16. Система уравнений

## Условие

Нужно найти unsigned 32-bit числа `x` и `y`, которые удовлетворяют системе:

```text
(a11 & x) ^ (a12 & y) = b1
(a21 & x) ^ (a22 & y) = b2
```

Если решений несколько, можно вывести любое.

Запрещены условные передачи и условное управление.

## Ввод

```text
a11 a12
a21 a22
b1  b2
```

Все числа unsigned 32-bit.

## Вывод

```text
x y
```

## Ограничения

- все операции битовые;
- решение гарантированно существует;
- нужно решить все 32 бита.

## Решение

Главная идея: каждый бит независим.

Для одного бита есть только 4 варианта:

```text
x=0 y=0
x=1 y=0
x=0 y=1
x=1 y=1
```

Но вместо цикла по битам можно построить 32-битные маски. Один бит маски равен `1`, если соответствующий кандидат подходит для этого бита.

## Маски кандидатов

Кандидат `x=0, y=0`:

```text
ok00 = ~(b1 | b2)
```

Кандидат `x=1, y=0`:

```text
ok10 = ~(a11 ^ b1) & ~(a21 ^ b2)
```

Кандидат `x=0, y=1`:

```text
ok01 = ~(a12 ^ b1) & ~(a22 ^ b2)
```

Кандидат `x=1, y=1`:

```text
ok11 = ~((a11 ^ a12) ^ b1) & ~((a21 ^ a22) ^ b2)
```

Почему работает: `p ^ q` даёт `0` там, где биты равны. После `not` получаем `1` на подходящих позициях.

## Важный шаг: сделать маски непересекающимися

Если для одного бита подходят несколько кандидатов, нельзя просто OR-ить всё подряд: можно случайно собрать новый кандидат, который не проверялся.

Поэтому выбираем первый подходящий кандидат по приоритету:

```text
m00 = ok00
m10 = ok10 & ~m00
m01 = ok01 & ~(m00 | m10)
m11 = ok11 & ~(m00 | m10 | m01)
```

Теперь каждый бит принадлежит ровно одному выбранному кандидату.

## Собираем ответ

`x` должен иметь `1` там, где выбран кандидат `10` или `11`:

```text
x = m10 | m11
```

`y` должен иметь `1` там, где выбран кандидат `01` или `11`:

```text
y = m01 | m11
```

## NASM-shape

Для каждого `ok` нужны только:

```asm
xor
not
and
or
```

Например:

```asm
; ok10 = ~(a11 ^ b1) & ~(a21 ^ b2)
mov eax, [a11]
xor eax, [b1]
not eax

mov ecx, [a21]
xor ecx, [b2]
not ecx

and eax, ecx
; eax = ok10
```

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| решать каждый бит через `if` | переходы запрещены |
| не разрулить пересечения `ok` | можно собрать невалидную пару `x/y` |
| забыть `not` после `xor` | получишь маску несовпадений, а нужна маска совпадений |
| перепутать `|` и `^` | система задана через `&` и `^` |
