# 01-8. Masked merge

## Коротко

Нужно для каждого бита выбрать бит из `a` или `b` по маске `c`.

<details open>
<summary>Подробное решение</summary>

Условие:

```text
d[i] = c[i] ? a[i] : b[i]
```

Если `c[i] = 1`, бит должен прийти из `a`.
Если `c[i] = 0`, бит должен прийти из `b`.

Формула:

```text
d = (a & c) | (b & ~c)
```

Почему:

| Часть | Что делает |
|---|---|
| `a & c` | оставляет в `a` только позиции, где `c=1` |
| `b & ~c` | оставляет в `b` только позиции, где `c=0` |
| `or` | склеивает две непересекающиеся части |

NASM-shape:

```asm
mov eax, [a]
and eax, [c]

mov ecx, [c]
not ecx
and ecx, [b]

or eax, ecx
```

Для ввода/вывода в hex обычно нужны `%x`:

```asm
fmtIn  db "%x %x %x", 0
fmtOut db "%x", 10, 0
```

</details>

## Ручные тесты

| `a` | `b` | `c` | Ответ |
|---|---|---|---|
| `1` | `0` | `1` | `1` |
| `2` | `1` | `2` | `3` |
| `12345678` | `AAAAAAAA` | `00000000` | `AAAAAAAA` |
| `12345678` | `AAAAAAAA` | `FFFFFFFF` | `12345678` |

## Где может сломаться

- написать `(a & ~c) | (b & c)`: получится выбор наоборот;
- использовать `%d` вместо `%x`;
- забыть, что `not ecx` меняет `ecx`, поэтому исходную `c` лучше отдельно загрузить.
