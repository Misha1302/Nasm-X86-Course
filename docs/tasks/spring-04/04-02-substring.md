# 04-2. Поиск подстроки в строке

## Условие

Даны две разные строки длиной не больше 1000.

Нужно проверить, является ли одна строка подстрокой другой.

- если первая строка — подстрока второй, вывести `1 2`;
- если вторая строка — подстрока первой, вывести `2 1`;
- иначе вывести `0`.

Запрещён `io.inc`. При вызове библиотечных функций стек должен быть выровнен по 16 байт.

## Ввод

```text
s1
s2
```

## Вывод

```text
1 2
```

или:

```text
2 1
```

или:

```text
0
```

## Ограничения

- каждая строка длиной не больше 1000;
- строки различны;
- нужно соблюдать 16-byte alignment перед libc calls.

## Решение

Используем `strstr`:

```c
strstr(haystack, needle)
```

Она возвращает не-null, если `needle` входит в `haystack`.

Проверки:

```text
if strstr(s2, s1) != NULL:
    print "1 2"
else if strstr(s1, s2) != NULL:
    print "2 1"
else:
    print "0"
```

## Память

```asm
section .bss
    s1 resb 1005
    s2 resb 1005
```

Формат чтения:

```asm
fmtStr db "%1000s", 0
```

## NASM-shape

Вызов `strstr(s2, s1)`:

```asm
push s1      ; needle
push s2      ; haystack
call strstr
add esp, 8

test eax, eax
jnz .first_inside_second
```

Перед реальным `call strstr` нужно соблюсти alignment.

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| перепутать порядок аргументов `strstr` | сначала haystack, потом needle |
| выделить ровно 1000 байт | нужен ещё `0`-terminator |
| читать через `%s` без ограничения | риск переполнения буфера |
| забыть alignment | по условию все библиотечные вызовы должны быть выровнены |
