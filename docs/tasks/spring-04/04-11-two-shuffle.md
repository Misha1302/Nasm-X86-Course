# 04-11. 2-перемешивание

## Коротко

Наивно двигать элементы массива нельзя: `N, M <= 100000`. Нужен linked list через массивы `prev[]` и `next[]`.

<details open>
<summary>Подробное решение</summary>

Изначальная последовательность:

```text
1 2 3 ... N
```

Операция `(b, c)`:

```text
вырезать сегмент от b до c и перенести его в начало
```

Порядок внутри сегмента не меняется.

### 1. Представление

Храним не сами элементы в массиве, а связи:

```text
next[value] = следующий элемент
prev[value] = предыдущий элемент
head = первый элемент
```

Инициализация:

```text
for i = 1..N:
    prev[i] = i - 1
    next[i] = i + 1

prev[1] = 0
next[N] = 0
head = 1
```

`0` — sentinel: нет элемента.

### 2. Операция cut + move to front

Дан сегмент `[b..c]` в текущем списке.

```text
before = prev[b]
after  = next[c]
```

Если `b == head`, сегмент уже начинается в голове. Делать ничего не нужно.

Иначе вырезаем сегмент:

```text
next[before] = after
if after != 0:
    prev[after] = before
```

Приклеиваем сегмент перед текущей головой:

```text
prev[head] = c
next[c] = head
prev[b] = 0
head = b
```

### 3. Почему это быстро

Каждая операция делает только несколько присваиваний:

```text
O(1)
```

Всего:

```text
O(N + M)
```

Это проходит для `100000`.

### 4. Вывод

Идём от `head` по `next`:

```text
cur = head
while cur != 0:
    print cur
    cur = next[cur]
```

## NASM-shape адресации

Если `next` и `prev` — массивы `dword`, а значение элемента лежит в `eax`:

```asm
mov ecx, [next + eax * 4]
mov edx, [prev + eax * 4]
```

Поскольку элементы нумеруются с `1`, нулевой элемент можно использовать как sentinel.

## Где может сломаться

- делать физический перенос массива за `O(NM)`;
- забыть обновить `prev[after]`;
- забыть случай `after = 0`;
- забыть случай `b = head`;
- испортить `head` до того, как старый `head` приклеен после `c`.
