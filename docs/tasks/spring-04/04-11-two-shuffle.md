# 04-11. 2-перемешивание

## Условие

Есть последовательность:

```text
1 2 3 ... N
```

`M` раз применяется операция: отрезок от элемента со значением `b` до элемента со значением `c` переносится в начало. Порядок внутри отрезка не меняется.

Гарантируется, что `b` в текущей последовательности стоит не позже `c`.

Вход из `input.txt`, выход в `output.txt`.

Запрещён `io.inc`. Перед libc calls нужен 16-byte alignment.

## Ввод

Файл `input.txt`:

```text
N M
b1 c1
b2 c2
...
bM cM
```

## Вывод

Файл `output.txt`: итоговая последовательность.

## Ограничения

- `N, M <= 100000`;
- наивный перенос массива `O(N*M)` не проходит.

## Решение

Нужен linked list через массивы.

Для каждого значения `v` храним:

```text
prev[v]
next[v]
```

Также храним:

```text
head
 tail
```

Начальное состояние:

```text
prev[1] = 0
next[1] = 2
...
prev[N] = N-1
next[N] = 0
head = 1
tail = N
```

## Операция move `[b..c]` to front

Пусть:

```text
before = prev[b]
after  = next[c]
```

Если `before == 0`, отрезок уже в начале, ничего делать не надо.

Иначе вырезаем отрезок:

```text
next[before] = after
if after != 0:
    prev[after] = before
else:
    tail = before
```

И вставляем в начало:

```text
prev[head] = c
next[c] = head
prev[b] = 0
head = b
```

## Почему это быстро

Каждая операция меняет только несколько ссылок:

```text
O(1) на один запрос
O(N + M) на всю задачу
```

## NASM-shape

Массивы:

```asm
prev resd 100001
next resd 100001
```

Доступ:

```asm
mov eax, [prev + 4*ebx]   ; prev[b]
mov ecx, [next + 4*edx]   ; next[c]
```

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| двигать элементы массива физически | `O(N*M)` не проходит |
| забыть обновить `tail`, если `after = 0` | потеряется конец списка |
| не обработать `before = 0` | можно сломать голову списка |
| перепутать значения и позиции | `b/c` — значения элементов, а не индексы массива |
| забыть файловый ввод/вывод | задача использует `input.txt` и `output.txt` |
