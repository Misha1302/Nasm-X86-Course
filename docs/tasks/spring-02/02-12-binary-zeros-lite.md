# 02-12. Двоичные нули лайт

## Условие

Нужно посчитать количество чисел от `1` до `N`, в двоичной записи которых ровно `K` значащих нулей.

Ведущие нули не считаются.

## Ввод

```text
N K
```

## Вывод

Одно число: количество подходящих чисел.

## Ограничения

- `0 <= N <= 100000`;
- `0 <= K <= 100000`.

## Решение

Так как `N` маленькое, можно перебрать все числа от `1` до `N`.

Для каждого числа считаем нули в его обычной двоичной записи.

Пример:

```text
18 = 10010b
значащие нули = 3
```

## Алгоритм

```text
answer = 0

for x = 1..N:
    t = x
    zeros = 0

    while t != 0:
        if (t & 1) == 0:
            zeros++
        t >>= 1

    if zeros == K:
        answer++

print answer
```

## NASM-shape

Проверка младшего бита:

```asm
test eax, 1
jnz .bit_is_one
inc zeros
.bit_is_one:
shr eax, 1
```

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| считать ведущие нули до 32 бит | нужны только значащие нули |
| начинать с `0` | диапазон от `1` до `N` |
| пытаться использовать этот перебор для 02-14 | там `N` до `10^9` |
| использовать signed shift | здесь достаточно `shr` |
