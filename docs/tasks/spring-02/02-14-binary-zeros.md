# 02-14. Двоичные нули

## Условие

Нужно посчитать количество чисел от `1` до `N`, в двоичной записи которых ровно `K` значащих нулей.

Это не lite-версия: `N` большое.

## Ввод

```text
N K
```

## Вывод

Одно число: количество подходящих чисел.

## Ограничения

- `0 <= N <= 1000000000`;
- `0 <= K <= 1000000000` по смыслу ввода, но реально больше 31 уже даёт 0;
- перебор `1..N` не проходит.

## Решение

Нужно считать не сами числа, а количество чисел по длине двоичной записи.

Если число имеет длину `len`, старший бит всегда `1`. Значащие нули находятся только в оставшихся `len - 1` позициях.

Количество `len`-битных чисел с ровно `K` нулями:

```text
C(len - 1, K)
```

## Алгоритм

### 1. Предпосчитать биномиальные коэффициенты

Достаточно таблицы до 31:

```text
C[n][0] = C[n][n] = 1
C[n][k] = C[n-1][k-1] + C[n-1][k]
```

Если `k < 0` или `k > n`, считаем `C(n,k) = 0`.

### 2. Добавить все длины меньше длины `N`

```text
answer = 0
L = bit_length(N)

for len = 1..L-1:
    answer += C(len - 1, K)
```

### 3. Посчитать числа той же длины, что `N`

Идём по битам `N` сверху вниз, кроме старшего бита.

Если в `N` текущий бит равен `1`, то можно поставить в ответном числе `0` на этом месте. Тогда число уже точно меньше `N`, а оставшиеся биты можно заполнить комбинаторикой.

```text
zerosUsed = 0
for pos = L-2 downto 0:
    if bit(N, pos) == 1:
        answer += C(pos, K - zerosUsed - 1)
    else:
        zerosUsed++

if zerosUsed == K:
    answer++      ; само N тоже подходит
```

## Пример идеи для `N = 18`

```text
18 = 10010b
```

Длины меньше 5 считаются сразу через `C`.

Для длины 5 идём по битам `0010` после старшей единицы и считаем варианты, где впервые делаем число меньше `N`.

## NASM-shape

Таблицу `C` можно хранить в `.bss`:

```asm
comb resd 32 * 32
```

Адрес:

```text
comb[n][k] = comb + 4 * (n * 32 + k)
```

Проверка бита:

```asm
mov eax, [N]
mov cl, [pos]
shr eax, cl
and eax, 1
```

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| перебор до `N` | при `10^9` не проходит |
| считать ведущие нули | они не значащие |
| забыть само `N` | если `N` подходит, его надо добавить |
| не обработать `K > 31` | ответ сразу 0 |
| неправильно считать `C(n,k)` для невозможных `k` | нужно возвращать 0 |
