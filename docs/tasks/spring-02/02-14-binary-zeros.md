# 02-14. Двоичные нули

## Коротко

`N` до `10^9`, поэтому перебор `1..N` не проходит. Нужно считать количество чисел по битам и биномиальным коэффициентам.

<details open>
<summary>Подробное решение</summary>

Нужно посчитать числа `1 <= x <= N`, у которых ровно `K` значащих нулей в двоичной записи.

Значащие нули — это нули после первой единицы.

Пример:

```text
100100₂
```

Длина `6`, единиц `2`, значащих нулей `4`.

### 1. Предпосчитать C(n, k)

Максимальная длина для `10^9` меньше `31` бит.

Достаточно таблицы:

```text
C[0..31][0..31]
```

Правило:

```text
C(n,0) = 1
C(n,n) = 1
C(n,k) = C(n-1,k-1) + C(n-1,k)
```

### 2. Числа меньшей длины

Для длины `len` старший бит всегда `1`.

Остальные `len - 1` битов могут содержать ровно `K` нулей.

Количество таких чисел:

```text
C(len - 1, K)
```

Суммируем для всех:

```text
len = 1..bit_length(N)-1
```

### 3. Числа той же длины, что N

Идём по битам `N` сверху вниз после старшего бита.

Пусть уже набрали префикс, равный префиксу `N`.

Если текущий бит `N` равен `1`, то можно поставить в ответе `0` на этой позиции. Тогда число уже станет меньше `N`, а оставшиеся биты можно заполнить как угодно.

Если ставим `0`, мы потратили один значащий ноль.

Добавляем:

```text
C(remainingBits, zerosLeft - 1)
```

После этого продолжаем путь, где текущий бит равен `1`, как в `N`.

Если текущий бит `N` равен `0`, мы обязаны поставить `0`, иначе превысим `N`. Значит:

```text
zerosLeft--
```

### 4. Учесть само N

После прохода всех битов, если:

```text
zerosLeft == 0
```

добавляем `1`, потому что само `N` подходит.

</details>

## Псевдокод

```text
answer = 0
L = bit_length(N)

for len in 1..L-1:
    answer += C(len - 1, K)

zerosLeft = K

for pos from L-2 downto 0:
    if bit(N, pos) == 1:
        answer += C(pos, zerosLeft - 1)
    else:
        zerosLeft--

    if zerosLeft < 0:
        break

if zerosLeft == 0:
    answer++
```

Для NASM `if` здесь разрешены: это обычная задача Spring-02, не branchless.

## Мини-тест на `N = 18`, `K = 3`

```text
18 = 10010₂
```

Подходящие числа:

```text
8  = 1000₂   -> 3 нуля
16 = 10000₂  -> 4 нуля, не подходит
17 = 10001₂  -> 3 нуля
18 = 10010₂  -> 3 нуля
```

Ответ `3`: `8`, `17`, `18`.

## Где может сломаться

- считать ведущие нули;
- забыть числа меньшей длины;
- забыть само `N`;
- обратиться к `C(n,k)` при `k < 0` или `k > n` — в таких случаях нужно считать `0`;
- пытаться перебрать до `10^9`.
