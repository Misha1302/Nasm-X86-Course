# 02-6. Поиск максимальной последовательности бит

## Условие

Дано unsigned 32-bit число `N` и число `K`.

Нужно выбрать `K` подряд идущих битов числа `N`, чтобы получившееся число было максимальным.

## Ввод

```text
N K
```

## Вывод

Одно число: максимальное значение среди всех `K`-битных окон.

## Ограничения

- `N` — unsigned 32-bit;
- `1 <= K <= 31`.

## Решение

Строим маску из `K` единиц:

```text
mask = (1 << K) - 1
```

Потом двигаем окно по всем возможным позициям:

```text
candidate = (N >> shift) & mask
```

Берём максимум.

## Алгоритм

```text
answer = 0
mask = (1 << K) - 1

for shift = 0..32-K:
    candidate = (N >> shift) & mask
    answer = max(answer, candidate)
```

Если `K` больше фактической длины числа, всё равно работает: младшее окно даст само число.

## NASM-shape

Переменный сдвиг использует `cl`:

```asm
mov eax, [N]
mov cl, [shift]
shr eax, cl
and eax, [mask]
```

Сравнение должно быть unsigned:

```asm
cmp eax, [answer]
jbe .not_better
mov [answer], eax
.not_better:
```

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| использовать signed `jg/jl` | число unsigned |
| сделать сдвиг не через `cl` | x86 требует `cl` для переменного shift |
| забыть проверить shift `32-K` включительно | последнее окно тоже нужно |
| делать `1 << 32` | `K <= 31`, но всё равно не выходи за границы |
