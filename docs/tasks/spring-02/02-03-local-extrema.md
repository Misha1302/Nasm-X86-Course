# 02-3. Локальные экстремумы

## Условие

Дана последовательность. Нужно вывести:

1. количество локальных минимумов и их индексы;
2. количество локальных максимумов и их индексы.

Локальный минимум строго меньше обоих соседей. Локальный максимум строго больше обоих соседей.

Первый и последний элементы не считаются экстремумами.

## Ввод

```text
N
a0 a1 ... aN-1
```

## Вывод

```text
m
indices_of_minima
M
indices_of_maxima
```

Если индексов нет, после количества просто печатается пустая строка.

## Ограничения

- `0 <= N <= 500000`;
- элементы — signed 32-bit.

## Решение

Не обязательно хранить весь массив. Достаточно держать три соседних значения:

```text
left, mid, right
```

Когда прочитали `right`, можно проверить `mid`.

Индекс `mid` равен `i - 1`, если `right` прочитан как элемент `i`.

## Алгоритм

```text
read N
if N < 3:
    print 0, empty line, 0, empty line

read left
read mid

for i = 2..N-1:
    read right

    if mid < left && mid < right:
        save i-1 to minimaIndices

    if mid > left && mid > right:
        save i-1 to maximaIndices

    left = mid
    mid = right

print minimaCount and minimaIndices
print maximaCount and maximaIndices
```

## NASM-shape

Для условия минимума:

```asm
; mid < left && mid < right
mov eax, [mid]
cmp eax, [left]
jge .not_min
cmp eax, [right]
jge .not_min
; save index
.not_min:
```

Для максимума аналогично через `jle`.

## Память

Нужно сохранить только индексы экстремумов:

```asm
minIdx resd 500000
maxIdx resd 500000
```

## Ошибки

| Ошибка | Почему плохо |
|---|---|
| проверять первый/последний элемент | у них нет двух соседей |
| использовать нестрогие сравнения | нужны строго меньше/строго больше |
| забыть вывести пустую строку для нулевого списка | формат может отличаться от ожидаемого |
| хранить все числа без необходимости | можно, но потоковая схема проще по памяти |
