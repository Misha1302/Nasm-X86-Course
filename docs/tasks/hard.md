# Подробные разборы сложных задач

Эти задачи стоит читать не как “готовые ответы”, а как набор архитектурных паттернов. После чтения ученик должен понимать, какой алгоритм писать на NASM.

| Задача | Почему сложная | Подробный разбор |

| 01-14 Огород | branchless-выбор + округления | [разбор](/tasks/spring-01/01-14-garden) |
| 01-16 Система | 32 битовые системы одновременно | [разбор](/tasks/spring-01/01-16-bit-system) |
| 02-14 Двоичные нули | комбинаторика по битам | [разбор](/tasks/spring-02/02-14-binary-zeros) |

| 04-13 Веселье со стеком | function pointer + varargs + alignment | [разбор](/tasks/spring-04/04-13-stack-fun) |

## Как проходить

1. Сначала прочитай “Коротко”.
2. Потом раскрой подробный разбор.
3. Перепиши C-shape на бумаге.
4. Только после этого переводи в NASM.

Если задача всё ещё кажется непонятной, значит не хватает паттерна:

| Не хватает | Куда идти |
|---|---|
| маски без `if` | [Branchless-маски](/patterns/branchless) |
| битовый подсчёт | [Битовые циклы](/patterns/bit_counting) |
| связный список массивами | [Массивная связность](/patterns/array_linked_list) |
| varargs/cdecl | [Advanced stack](/patterns/advanced_stack) |
| большое число | [Big integer](/patterns/bigint) |
