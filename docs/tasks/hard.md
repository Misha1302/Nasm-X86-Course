# Подробные разборы сложных задач

Эта страница — быстрый маршрут по задачам, где одной базовой главы мало. Здесь есть не готовый NASM-код, а полный план решения: что хранить, какой алгоритм использовать и где чаще всего ломается реализация.

## 01-14. Огород

**Почему сложная:** условие запрещает переходы, но нужно выбрать дневной или ночной сценарий.

**Ключ:** сначала посчитать оба ответа, потом выбрать через маску.

```text
items = N * M * K
boxes = ceil(items / D)
blocked = ceil(boxes / 3)

dayAnswer = boxes - blocked
nightAnswer = boxes
mask = -1 if X < 6 else 0
answer = (nightAnswer & mask) | (dayAnswer & ~mask)
```

Подробно: [01-14 Огород](./spring-01/01-14-garden).

## 01-16. Система уравнений

**Почему сложная:** это не обычная система чисел, а 32 независимые системы над битами.

**Ключ:** построить маски для 4 кандидатов `(x,y)` и сделать их непересекающимися.

```text
ok00 = ~(b1 | b2)
ok10 = ~(a11 ^ b1) & ~(a21 ^ b2)
ok01 = ~(a12 ^ b1) & ~(a22 ^ b2)
ok11 = ~((a11 ^ a12) ^ b1) & ~((a21 ^ a22) ^ b2)
```

Потом:

```text
m00 = ok00
m10 = ok10 & ~m00
m01 = ok01 & ~(m00 | m10)
m11 = ok11 & ~(m00 | m10 | m01)

x = m10 | m11
y = m01 | m11
```

Подробно: [01-16 Система уравнений](./spring-01/01-16-bit-system).

## 02-14. Двоичные нули

**Почему сложная:** `N` до `10^9`, перебор не проходит.

**Ключ:** считать числа по длине и по префиксу через `C(n,k)`.

```text
for len = 1..bit_length(N)-1:
    answer += C(len - 1, K)
```

Для длины как у `N` идём сверху вниз. Когда у `N` стоит `1`, можно поставить `0`, стать меньше `N` и добрать остаток комбинаторикой.

Подробно: [02-14 Двоичные нули](./spring-02/02-14-binary-zeros).

## 03-18. Знаковое произведение

**Почему сложная:** произведение трёх signed 32-bit чисел может не поместиться в 64 бита.

**Ключ:** знак отдельно, модуль результата как 96-bit `hi:mid:lo`, печать через деление большого числа на 10.

```text
sign = sign(a) xor sign(b) xor sign(c)
P = abs(a) * abs(b) * abs(c)   ; 96-bit
print_decimal(P)
```

Подробно: [03-18 Знаковое произведение](./spring-03/03-18-signed-product).

## 04-11. 2-перемешивание

**Почему сложная:** `N,M <= 100000`, физический перенос массива не проходит.

**Ключ:** linked list через массивы `prev[value]` и `next[value]`.

Операция `[b..c] -> front` меняет только несколько ссылок:

```text
before = prev[b]
after = next[c]
```

Вырезать сегмент и вставить перед текущим `head`.

Подробно: [04-11 2-перемешивание](./spring-04/04-11-two-shuffle).

## 04-13. Веселье со стеком

**Почему сложная:** нужно реализовать `apply` с function pointer и varargs.

**Ключ:** `apply` копирует varargs на стек для каждого вызова `fn`, добавляя текущий элемент массива последним аргументом.

```text
push array[i]
for k = n-1 downto 0:
    push vararg[k]
call fn
```
