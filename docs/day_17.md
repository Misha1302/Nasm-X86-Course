## Зачем этот день

`call` уже понятен. Теперь главный вопрос: как функция узнаёт, где её аргументы и локальные переменные? Ответ: через фрейм и `ebp`.

## Главная мысль

В CDECL аргументы лежат выше `ebp`, локальные переменные ниже `ebp`, результат возвращается в `eax`.

## Классический пролог и эпилог

```asm
push ebp
mov ebp, esp
sub esp, N

; body

mov esp, ebp
pop ebp
ret
```

Иногда вместо последних двух строк используют:

```asm
leave
ret
```

## Карта фрейма

```text
higher addresses
+------------------+
| argument 2       | [ebp+12]
+------------------+
| argument 1       | [ebp+8]
+------------------+
| return address   | [ebp+4]
+------------------+
| old ebp          | [ebp]
+------------------+
| local variable 1 | [ebp-4]
+------------------+
| local variable 2 | [ebp-8]
+------------------+
lower addresses
```

Первый аргумент — `[ebp+8]`, потому что `[ebp]` занят старым `ebp`, а `[ebp+4]` — адрес возврата.

## Пример функции `sum`

```asm
sum:
    push ebp
    mov ebp, esp

    mov eax, [ebp+8]
    add eax, [ebp+12]

    pop ebp
    ret
```

Вызов:

```asm
push dword [b]
push dword [a]
call sum
add esp, 8
```

## Caller-saved / callee-saved

| Register | Кто отвечает |
|---|---|
| `eax`, `ecx`, `edx` | caller должен не надеяться, что они сохранятся |
| `ebx`, `esi`, `edi` | callee должен восстановить, если использовал |
| `esp`, `ebp` | должны быть корректно восстановлены |

## Мини-челленджи

1. Где второй аргумент функции?
2. Почему локальные переменные имеют отрицательные смещения?
3. Напиши `square(x)`.
4. Почему после `call sum` нужно `add esp, 8`?

<details>
<summary>Ответы / подсказки</summary>

1. `[ebp+12]`.
2. Стек растёт вниз, локальные выделяются ниже текущего `ebp`.
3. `mov eax,[ebp+8]`; `imul eax,eax`.
4. Caller чистит два аргумента по 4 байта.

</details>
