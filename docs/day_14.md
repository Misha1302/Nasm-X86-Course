# День 14. `switch`, jump table и fall-through

## Опора на материалы ВШЭ

`Slides2026-10.pdf`, `Slides2026-0xB.pdf`: `switch`, цепочки сравнений, таблицы переходов, fall-through, Duff’s Device и обратные задачи по восстановлению C-кода из asm.

## Зачем этот день

`if` и `while` обычно легко узнать по `cmp`, `jcc`, `jmp`.

А вот `switch` иногда выглядит странно:

```asm
jmp [.table + 4*eax]
```

Это не обычный переход на одну метку. Это переход по адресу, который лежит в таблице.

На экзамене такой код часто дают как reverse-задачу: нужно понять, какие были `case`, где `default`, какие ветки объединены и где есть fall-through.

---

## Главная мысль

Jump table — это массив адресов.

Индекс выбирает, на какую метку прыгнуть.

```text
index = value - minCase
jump to table[index]
```

В IA-32 адрес занимает 4 байта, поэтому часто видно:

```asm
jmp [.table + 4*eax]
```

---

# 1. `switch` как цепочка сравнений

Для маленького `switch` компилятор может сделать просто много сравнений.

C++:

```cpp
switch (x) {
    case 1: a = 10; break;
    case 2: a = 20; break;
    default: a = 0; break;
}
```

ASM-shape:

```asm
cmp eax, 1
je .case1
cmp eax, 2
je .case2
jmp .default

.case1:
    mov dword [a], 10
    jmp .end

.case2:
    mov dword [a], 20
    jmp .end

.default:
    mov dword [a], 0

.end:
```

Это похоже на `if/else if/else`.

---

# 2. `switch` как jump table

Если `case` идут плотным диапазоном, компилятору выгодно сделать таблицу адресов.

C++:

```cpp
switch (x) {
    case 0: goto case0;
    case 1: goto case1;
    case 2: goto case2;
    case 3: goto case3;
    default: goto def;
}
```

ASM-shape:

```asm
cmp eax, 3
ja .default
jmp [.table + 4*eax]

.table:
    dd .case0
    dd .case1
    dd .case2
    dd .case3
```

Разбор:

```asm
cmp eax, 3
ja .default
```

Если `eax > 3` как unsigned, значит индекс вне диапазона `0..3`, идём в default.

```asm
jmp [.table + 4*eax]
```

Берём адрес из таблицы:

```text
.table + 4*eax
```

Почему `4`? Потому что адрес в IA-32 занимает 4 байта.

---

# 3. Как выглядит таблица

```asm
.table:
    dd .case0
    dd .case1
    dd .case2
    dd .case3
```

Это почти как массив:

```cpp
void* table[] = { case0, case1, case2, case3 };
```

Если:

```text
eax = 2
```

то:

```text
.table + 4*2 = address of table[2]
```

В `table[2]` лежит адрес `.case2`, значит процессор прыгнет в `.case2`.

---

# 4. Почему используется unsigned-проверка `ja`

Часто перед таблицей видим:

```asm
cmp eax, max
ja .default
```

`ja` — unsigned `>`.

Это удобно: если `eax` вдруг отрицательный как signed, то как unsigned он станет огромным числом и тоже уйдёт в default.

Например:

```text
eax = -1 = FFFFFFFFh
```

Как unsigned это `4294967295`, значит точно `above max`.

---

# 5. Диапазон не с нуля

C++:

```cpp
switch (x) {
    case -2: ...
    case -1: ...
    case 0: ...
    case 1: ...
    case 2: ...
    case 3: ...
    case 4: ...
}
```

Чтобы использовать таблицу, компилятор может сдвинуть значение:

```asm
add eax, 2       ; eax = x + 2
cmp eax, 6       ; индексы 0..6
ja .default
jmp [.table + 4*eax]
```

Почему `+2`?

Чтобы:

```text
x = -2 -> index = 0
x = -1 -> index = 1
x =  0 -> index = 2
...
x =  4 -> index = 6
```

# 6. Разбор экзаменационного примера

```asm
add eax, 2
cmp eax, 6
ja .L2
jmp [.L8 + 4*eax]

section .rodata
.L8 dd .L3, .L2, .L4, .L5, .L6, .L6, .L7
```

Читаем по шагам.

## Шаг 1. Сдвиг диапазона

```asm
add eax, 2
```

Значит, исходный `x` превратили в `x + 2`.

Если индекс должен быть `0..6`, то исходные `x` были:

```text
x = -2..4
```

## Шаг 2. Проверка диапазона

```asm
cmp eax, 6
ja .L2
```

Если индекс больше 6 — default или особая ветка `.L2`.

## Шаг 3. Таблица

```asm
.L8 dd .L3, .L2, .L4, .L5, .L6, .L6, .L7
```

Таблица по индексам:

| Индекс | Исходный x | Метка |
|---:|---:|---|
| 0 | -2 | `.L3` |
| 1 | -1 | `.L2` |
| 2 | 0 | `.L4` |
| 3 | 1 | `.L5` |
| 4 | 2 | `.L6` |
| 5 | 3 | `.L6` |
| 6 | 4 | `.L7` |

`.L6` встречается два раза, значит `case 2` и `case 3` ведут в один код.

`.L2` используется и при выходе за диапазон, и для индекса 1. Это может означать, что `case -1` ведёт туда же, куда `default`.

# 7. Fall-through

В C/C++ у `switch` есть особенность: если не написать `break`, выполнение может провалиться в следующий `case`.

```cpp
switch (x) {
    case 1:
        a += 10;
        // no break
    case 2:
        a += 20;
        break;
}
```

В asm это может выглядеть как отсутствие `jmp .end` в конце одной ветки:

```asm
.case1:
    add dword [a], 10
    ; no jump here

.case2:
    add dword [a], 20
    jmp .end
```

Если видишь, что ветка не заканчивается переходом к `.end`, а код просто продолжается следующей меткой — это может быть fall-through.

---

# 8. Duff’s Device — зачем его вообще показывают

Duff’s Device — это странный C-трюк, где `switch` смешивается с циклом и fall-through.

Тебе не нужно писать такое в задачах. Важно понять идею:

- `case` в C — это метки внутри блока;
- выполнение может “проваливаться” дальше;
- поэтому `switch` не всегда похож на аккуратный набор независимых веток.

Для курса это нужно не как стиль программирования, а как способ понять, насколько напрямую C-конструкции могут отображаться в переходы.

# 9. Паттерны для reverse engineering

| Pattern | Meaning |
|---|---|
| `cmp eax, max` + `ja default` | проверка выхода за диапазон |
| `sub eax, min` или `add eax, const` | сдвиг диапазона case к нулю |
| `jmp [table + 4*eax]` | переход по jump table |
| одинаковая метка в таблице | несколько `case` ведут в одну ветку |
| default-метка в таблице | некоторый `case` объединён с default |
| нет `jmp .end` в конце case | возможно fall-through |
| таблица в `.rodata` | массив адресов/констант |

# 10. Мини-челленджи

### 1. Диапазон case

```asm
sub eax, 10
cmp eax, 4
ja .default
jmp [.table + 4*eax]
```

Какой диапазон исходных `x` покрывает таблица?

<details>
<summary>Ответ</summary>

После `sub eax, 10` индекс равен `x - 10`. Индексы `0..4`, значит исходные `x = 10..14`.

</details>

### 2. Повтор метки

```asm
.table:
    dd .A, .B, .B, .C
```

Что значит повтор `.B`?

<details>
<summary>Ответ</summary>

Два разных `case` ведут в один и тот же код.

</details>

### 3. Почему `4*eax`?

<details>
<summary>Ответ</summary>

Потому что в IA-32 адрес занимает 4 байта, а таблица — массив адресов.

</details>

### 4. Найди исходные `x`

```asm
add eax, 3
cmp eax, 5
ja .D
jmp [.T + 4*eax]
```

<details>
<summary>Ответ</summary>

Индексы `0..5`. Раз `index = x + 3`, то `x = -3..2`.

</details>

### 5. Где может быть fall-through?

```asm
.case1:
    add eax, 10
.case2:
    add eax, 20
    jmp .end
```

<details>
<summary>Ответ</summary>

Между `.case1` и `.case2`: после `.case1` нет перехода к `.end`, поэтому выполнение проваливается в `.case2`.

</details>

# 11. Типовые ошибки

| Ошибка | Почему плохо |
|---|---|
| думать, что jump table — это массив значений `case` | это массив адресов перехода |
| забыть сдвиг диапазона | неправильно восстановишь значения `case` |
| перепутать индекс и исходное значение | `eax` после `add/sub` уже не исходный `x` |
| игнорировать повторяющиеся метки | потеряешь объединённые cases |
| не заметить fall-through | восстановишь лишний `break` |
| забыть, что `ja` unsigned | отрицательные signed-значения тоже могут уйти в default после unsigned-проверки |
