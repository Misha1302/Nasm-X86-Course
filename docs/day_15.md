## Зачем этот день

`a[i]` в C++ выглядит просто. В asm это адресная формула. Если понять `[base+index*scale+offset]`, массивы и структуры становятся намного проще.

## Главная мысль

`mov` с квадратными скобками читает память. `lea` только считает адрес или арифметическое выражение.

## Общая форма адреса

```text
[ base + index * scale + displacement ]

Example:
[eax + ecx*4 + 8]

base = eax
index = ecx
scale = 4
displacement = 8
```

`scale` может быть 1, 2, 4 или 8.

## `mov` против `lea`

```asm
mov eax, [edx + 4*ecx]   ; eax = a[i]
lea eax, [edx + 4*ecx]   ; eax = &a[i]
```

Таблица:

| Инструкция | Читает память? | Смысл |
|---|---:|---|
| `mov eax, [addr]` | да | загрузить значение из памяти |
| `lea eax, [addr]` | нет | посчитать адрес/выражение |

## Массивы

```cpp
int a[5];
```

```text
Address:
a+0    a+4    a+8    a+12   a+16
+------+------+------+------+------+
| a[0] | a[1] | a[2] | a[3] | a[4] |
+------+------+------+------+------+
```

Адрес `a[i]`:

```text
base + 4*i
```

## 2D-массивы

C/C++ хранит массивы row-major:

```cpp
int a[R][C];
```

```text
&a[i][j] = base + 4 * (i*C + j)
```

3. Запиши адрес `int a[i][j]`, если в строке `C` столбцов.
4. Что может означать `+8` в `[eax+4*ecx+8]`?

<details>
<summary>Ответы / подсказки</summary>

1. `mov` читает значение из памяти, `lea` считает адрес.
2. `base + 2*i`.
3. `base + 4*(i*C+j)`.
4. Смещение поля, заголовок структуры, пропуск до массива внутри структуры и т.п.
