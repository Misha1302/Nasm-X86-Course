# День 15. Адресация и `lea`: массивы без магии

## Опора на материалы ВШЭ

`Slides2026-05.pdf`, `Slides2026-10.pdf`: виды адресации, `LEA`, массивы, row-major layout, оптимизация доступа к массивам и обратные задачи по адресным формулам.

## Зачем этот день

В C++ ты пишешь просто:

```cpp
a[i]
a[i][j]
s.field
```

Но процессор не знает ни `i`, ни `field`, ни “двумерный массив”. Он знает адреса.

Сегодня мы разбираем главную формулу памяти в x86:

```asm
[base + index * scale + displacement]
```

Если она станет понятной, массивы, структуры и reverse engineering станут намного проще.

---

## Главная мысль

Квадратные скобки означают обращение к памяти.

`lea` считает адрес, но память не читает.

```asm
mov eax, [edx + 4*ecx]   ; прочитать значение по адресу
lea eax, [edx + 4*ecx]   ; посчитать сам адрес
```

---

# 1. Общая форма адреса

В x86 можно адресовать память так:

```text
[ base + index * scale + displacement ]
```

Пример:

```asm
[eax + ecx*4 + 8]
```

Разбор:

| Часть | Значение |
|---|---|
| base | `eax` |
| index | `ecx` |
| scale | `4` |
| displacement | `8` |

То есть адрес равен:

```text
eax + ecx * 4 + 8
```

`scale` может быть только:

```text
1, 2, 4, 8
```

Почему именно такие? Потому что это удобно для размеров элементов массивов: `char`, `short`, `int`, `long long`.

---

# 2. Простые формы адресации

| Форма | Пример | Где встречается |
|---|---|---|
| `[base]` | `[eax]` | указатель |
| `[base + disp]` | `[ebp-4]` | локальная переменная/поле |
| `[base + index]` | `[eax+ecx]` | byte-массив |
| `[base + index*scale]` | `[eax+4*ecx]` | `int a[i]` |
| `[base + index*scale + disp]` | `[eax+4*ecx+8]` | массив внутри структуры |

Пример:

```asm
mov eax, [ebp-4]
```

может означать “прочитать локальную переменную”.

```asm
mov eax, [edx+4*ecx]
```

может означать “прочитать `int`-элемент массива”.

---

# 3. `mov` против `lea`

Это одна из главных ловушек.

```asm
mov eax, [edx + 4*ecx]
```

значит:

```cpp
eax = *(int*)(edx + 4*ecx);
```

То есть память читается.

А:

```asm
lea eax, [edx + 4*ecx]
```

значит:

```cpp
eax = edx + 4*ecx;
```

Память не читается.

Таблица:

| Инструкция | Читает память? | Смысл |
|---|---:|---|
| `mov eax, [addr]` | да | загрузить значение из памяти |
| `lea eax, [addr]` | нет | посчитать адрес/арифметику |

---

## Почему `lea` называется странно

`lea` = load effective address.

Название может сбить с толку: “load” звучит так, будто мы что-то читаем из памяти.

Но на практике:

```asm
lea eax, [ebx + 4*ecx + 8]
```

просто считает:

```text
eax = ebx + 4*ecx + 8
```

Это можно использовать не только для адресов, но и как удобную арифметику:

```asm
lea eax, [eax + 2*eax] ; eax = eax * 3
```

# 4. Одномерный массив

C++:

```cpp
int a[5];
```

В памяти:

```text
a+0    a+4    a+8    a+12   a+16
+------+------+------+------+------+
| a[0] | a[1] | a[2] | a[3] | a[4] |
+------+------+------+------+------+
```

Почему шаг 4?

Потому что `int` занимает 4 байта.

Адрес:

```text
&a[i] = base + 4*i
```

Если:

```text
edx = base address of a
ecx = i
```

то:

```asm
mov eax, [edx + 4*ecx] ; eax = a[i]
lea eax, [edx + 4*ecx] ; eax = &a[i]
```

---

# 5. Массивы разных типов

| C/C++ type | Size | Address of `arr[i]` |
|---|---:|---|
| `char` | 1 | `base + i` |
| `short` | 2 | `base + 2*i` |
| `int` | 4 | `base + 4*i` |
| `long long` | 8 | `base + 8*i` |

Примеры:

```asm
; char c[i]
movzx eax, byte [edx + ecx]

; short s[i]
movsx eax, word [edx + 2*ecx]

; int a[i]
mov eax, [edx + 4*ecx]

; long long q[i], младшая/старшая часть отдельно в IA-32
mov eax, [edx + 8*ecx]
mov edx, [edx + 8*ecx + 4]
```

Для `char`/`short` часто нужны `movsx` или `movzx`, потому что читаем меньше 32 бит.

# 6. Двумерный массив: row-major

C/C++ хранит двумерные массивы по строкам.

```cpp
int a[R][C];
```

Логически:

```text
row 0: a[0][0] a[0][1] a[0][2]
row 1: a[1][0] a[1][1] a[1][2]
```

В памяти:

```text
a[0][0] a[0][1] a[0][2] a[1][0] a[1][1] a[1][2]
```

Адрес:

```text
&a[i][j] = base + sizeof(int) * (i*C + j)
```

Для `int`:

```text
&a[i][j] = base + 4 * (i*C + j)
```

Если `C = 10`:

```text
&a[i][j] = base + 4 * (10*i + j)
```

---

## Пример 2D

C++:

```cpp
int a[3][5];
int x = a[i][j];
```

Формула:

```text
address = base + 4 * (i*5 + j)
```

ASM-идея:

```asm
mov eax, [i]
imul eax, 5
add eax, [j]
mov eax, [base + 4*eax]
```

Если base в `edx`:

```asm
mov eax, [i]
imul eax, 5
add eax, [j]
mov eax, [edx + 4*eax]
```

---

# 7. Почему проход по строкам быстрее прохода по столбцам

Для C/C++ row-major массива:

```cpp
int a[R][C];
```

элементы одной строки лежат рядом.

Хорошо:

```cpp
for (int i = 0; i < R; ++i)
    for (int j = 0; j < C; ++j)
        sum += a[i][j];
```

Плохо для cache locality:

```cpp
for (int j = 0; j < C; ++j)
    for (int i = 0; i < R; ++i)
        sum += a[i][j];
```

Почему?

Первый вариант идёт по памяти почти подряд.

Второй прыгает через целые строки.

На уровне адресов это видно напрямую:

```text
a[i][j+1] рядом с a[i][j]
a[i+1][j] далеко на C элементов
```

# 8. Адресация внутри структуры

Пока структуры подробно будут в дне 19, но адресная формула уже готовит к ним.

```asm
[eax + 4*ecx + 8]
```

может означать:

```cpp
obj->a[i]
```

если:

```text
eax = address of obj
8   = offset of field a
4   = sizeof(int)
ecx = i
```

То есть `+8` часто значит “смещение поля внутри структуры”.

---

# 9. Reverse-задачи по формулам адреса

Если видишь:

```asm
mov eax, [edx + 4*ecx]
```

возможный смысл:

```cpp
eax = a[i]; // int array
```

Если видишь:

```asm
movzx eax, byte [edx + ecx]
```

возможный смысл:

```cpp
eax = (unsigned char)c[i];
```

Если видишь:

```asm
movsx eax, word [edx + 2*ecx]
```

возможный смысл:

```cpp
eax = (int)(short)s[i];
```

Если видишь:

```asm
lea eax, [edx + 4*ecx]
```

возможный смысл:

```cpp
eax = &a[i];
```

# 10. Мини-челленджи

### 1. `mov` или `lea`

Чем отличаются?

```asm
mov eax, [edx + 4*ecx]
lea eax, [edx + 4*ecx]
```

<details>
<summary>Ответ</summary>

`mov` читает значение из памяти по адресу `edx + 4*ecx`.

`lea` просто считает адрес `edx + 4*ecx`.

</details>

### 2. Адрес `short s[i]`

Если `edx = base`, `ecx = i`, какой адрес?

<details>
<summary>Ответ</summary>

```asm
[edx + 2*ecx]
```

</details>

### 3. Адрес `int a[i][j]`

В строке `C` столбцов. Формула?

<details>
<summary>Ответ</summary>

```text
base + 4 * (i*C + j)
```

</details>

### 4. Что может означать `+8`?

```asm
[eax + 4*ecx + 8]
```

<details>
<summary>Ответ</summary>

Например, смещение поля-массива внутри структуры: `obj->a[i]`, где `a` начинается с offset 8.

</details>

### 5. Как прочитать `unsigned char c[i]`?

<details>
<summary>Ответ</summary>

```asm
movzx eax, byte [base + i]
```

Например:

```asm
movzx eax, byte [edx + ecx]
```

</details>

# 11. Типовые ошибки

| Ошибка | Почему плохо |
|---|---|
| думать, что `lea` читает память | `lea` только считает адрес |
| забыть умножение на размер элемента | `int a[i]` — это `base + 4*i` |
| перепутать row-major и column-major | формула `i*C+j`, не `j*R+i` |
| использовать `mov eax, [charArray+i]` без `byte/movzx` | прочитаешь лишние байты |
| не учитывать `+offset` поля структуры | попадёшь не в тот field |
| забыть, что scale только 1/2/4/8 | нельзя напрямую `[eax+3*ecx]` |
