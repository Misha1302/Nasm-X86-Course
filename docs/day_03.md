## Зачем этот день

Нужно перестать думать “процессор понимает программу”. Нет, процессор выполняет маленькие команды одну за другой.

## Главная мысль

Инструкция = операция + операнды. Операнд может быть регистром, памятью или константой.

```text
+---------------- CPU ----------------+
| eax ebx ecx edx esi edi ebp esp     |
| eip     -> next instruction          |
| eflags  -> flags after operations    |
| ALU     -> arithmetic and logic      |
+------------------|------------------+
                   v
+---------------- RAM ----------------+
| code | data | heap | stack | ...    |
+--------------------------------------+
```

## Как процессор выполняет инструкцию

Упрощённо:

1. взять байты инструкции по адресу `eip`;
2. понять, что это за команда;
3. найти операнды;
4. выполнить действие;
5. записать результат;
6. перейти к следующей инструкции.

## Разбор инструкций

```asm
mov eax, ecx              ; register <- register
mov ecx, dword [0x2019]   ; register <- memory
mov dword [ebp-4], ebx    ; memory <- register
add eax, 0xff             ; register += immediate
```

| Операнд | Пример | Как думать |
|---|---|---|
| register | `eax` | быстрая ячейка внутри CPU |
| memory | `[ebp-4]` | значение по адресу |
| immediate | `0xff` | константа прямо в инструкции |

## Важный момент про `mov`

`mov` не “перемещает” как физическое перемещение. Он **копирует**.

```asm
mov eax, ebx
```

После этого `eax` получил значение `ebx`, но `ebx` не очистился.

## Мини-челленджи

1. В `add eax, ebx` кто destination, кто source?
2. Почему `add eax, ebx` ближе к `eax += ebx`, а не к `eax + ebx`?
3. Для `mov [ebp-4], ebx` что является памятью?

<details>
<summary>Ответы / подсказки</summary>

1. Destination — `eax`, source — `ebx`.
2. Результат записывается обратно в `eax`.
3. `[ebp-4]` — ячейка памяти по адресу `ebp-4`.

</details>

## Что должно остаться в голове

Процессор не видит красивый C++-код. Он видит команды, операнды, регистры, память и адрес следующей инструкции.
