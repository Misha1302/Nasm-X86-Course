# День 03. Процессор как тупая, но очень быстрая машина

## Опора на материалы ВШЭ

`Slides2026-02.pdf`, `Slides2026-03.pdf`: цикл работы ЭВМ, операнды, инструкции, effective address.

## Зачем этот день

Перед тем как писать NASM, нужно договориться о модели мира.

В C++ ты думаешь примерно так:

```cpp
int x = a + b * 2;
```

А процессор не видит “выражение”. Он видит очень простые команды и выполняет их одну за другой.

Сегодня мы разбираем три главные вещи:

1. где процессор хранит временные значения;
2. откуда он берёт данные;
3. из чего состоит инструкция.

---

## Главная мысль

Инструкция — это операция плюс операнды.

Операнд может быть:

- регистром;
- памятью;
- константой прямо внутри инструкции.

```text
+---------------- CPU ----------------+
|                                      |
|  registers: eax ebx ecx edx ...     |
|  eip: address of next instruction    |
|  eflags: flags after operations      |
|  ALU: arithmetic and logic           |
|                                      |
+------------------|------------------+
                   |
                   v
+---------------- RAM ----------------+
| code | data | heap | stack | ...    |
+--------------------------------------+
```

Самая простая картина:

```text
memory  <->  registers  <->  ALU
```

Данные часто лежат в памяти, но считать удобно в регистрах.

---

## Процессор не “понимает программу”

Он делает цикл:

```text
fetch   -> взять байты инструкции по адресу eip
decode  -> понять, какая это инструкция
execute -> выполнить действие
write   -> записать результат
next    -> перейти к следующей инструкции
```

Упрощённо:

```text
eip указывает на инструкцию
CPU читает инструкцию
CPU выполняет её
eip переходит дальше
```

Позже мы увидим, что `jmp`, `call`, `ret` и условные переходы меняют этот обычный порядок.

---

## Что такое регистр

Регистр — это маленькая быстрая ячейка внутри процессора.

Можно думать так:

```text
память  = огромная тетрадь;
регистр = маленький листочек прямо в руках у процессора.
```

Пример:

```asm
mov eax, 10
add eax, 5
```

После этого:

```text
eax = 15
```

Мы не ходили в память. Всё было в регистре.

---

## Виды операндов

| Вид операнда | Пример | Как думать |
|---|---|---|
| register | `eax` | значение внутри CPU |
| memory | `[ebp-4]` | значение в RAM по адресу |
| immediate | `0xff` | число прямо в инструкции |

Разберём строки:

```asm
mov eax, ecx              ; register <- register
mov ecx, dword [0x2019]   ; register <- memory
mov dword [ebp-4], ebx    ; memory <- register
add eax, 0xff             ; register += immediate
```

### `mov eax, ecx`

```text
eax = ecx
```

Значение копируется из одного регистра в другой.

### `mov ecx, dword [0x2019]`

```text
ecx = memory[0x2019..0x201C]
```

`dword` говорит, что читаем 4 байта.

### `mov dword [ebp-4], ebx`

```text
memory[ebp-4] = ebx
```

Это запись в память.

### `add eax, 0xff`

```text
eax = eax + 0xff
```

Первый операнд меняется.

---

## Важное правило порядка операндов

В NASM обычно порядок такой:

```asm
instruction destination, source
```

То есть:

```asm
mov eax, ebx    ; eax = ebx
add eax, ebx    ; eax = eax + ebx
sub eax, 10     ; eax = eax - 10
```

Это не универсальная истина для всех ассемблерных синтаксисов. Например, AT&T-синтаксис пишет иначе. Но в этом курсе мы используем NASM/Intel-style, поэтому держим в голове именно это.

---

## `mov` копирует, а не переносит

Название `mov` слегка обманывает. Оно звучит как “переместить”, но на самом деле это копирование.

```asm
mov eax, ebx
```

После этой команды:

```text
eax получил значение ebx
ebx остался прежним
```

C++-аналогия:

```cpp
eax = ebx;
```

А не:

```cpp
eax = ebx;
ebx = 0;
```

---

## Почему нельзя память-память

В x86 многие инструкции не позволяют напрямую делать:

```asm
mov [a], [b] ; плохо / нельзя в таком виде
```

Обычно нужно через регистр:

```asm
mov eax, [b]
mov [a], eax
```

Это хороший стиль мышления:

```text
память -> регистр -> память
```

Регистры — рабочее место процессора.

---

## Мини-разбор C++ выражения

C++:

```cpp
x = a + b * 2;
```

Псевдо-NASM:

```asm
mov eax, [b]    ; eax = b
imul eax, 2     ; eax = b * 2
add eax, [a]    ; eax = a + b * 2
mov [x], eax    ; x = eax
```

Обрати внимание: мы не считаем всё выражение “одной магией”. Мы выбираем временный регистр и делаем шаги.

---

## Что такое `eip`

`eip` — instruction pointer. Он хранит адрес следующей инструкции.

Обычно после инструкции `eip` просто двигается дальше.

```text
instruction 1  <- eip
instruction 2
instruction 3
```

После выполнения первой:

```text
instruction 1
instruction 2  <- eip
instruction 3
```

Но переходы меняют `eip` не “на следующую”, а на заданную метку.

```asm
jmp .loop
```

Это значит: следующая инструкция будет не та, что лежит ниже, а та, что находится по адресу `.loop`.

---

## Что такое `eflags`

`eflags` — набор флагов, которые описывают результат некоторых операций.

Например:

```asm
sub eax, eax
```

Результат стал ноль. Значит, включится `ZF` — zero flag.

Пока не надо знать все флаги. Просто запомни:

```text
операции могут оставлять после себя “следы” во флагах;
условные переходы потом смотрят на эти следы.
```

Подробно разберём это в днях 11–12.

## Мини-челленджи

### 1. Destination и source

В строке:

```asm
add eax, ebx
```

кто destination, кто source?

<details>
<summary>Ответ</summary>

Destination — `eax`. Source — `ebx`. Результат записывается в `eax`.

</details>

### 2. Перевод в C++

Переведи:

```asm
mov eax, [a]
sub eax, 5
mov [b], eax
```

<details>
<summary>Ответ</summary>

Примерный C++-смысл:

```cpp
b = a - 5;
```

</details>

### 3. Почему не `mov [a], [b]`?

Объясни идею.

<details>
<summary>Ответ</summary>

Многие x86-инструкции не работают напрямую память-память. Нужно загрузить значение из памяти в регистр, а потом записать из регистра в память.

</details>

### 4. Что делает `mov`?

Выбери правильное:

```text
A. переносит значение и очищает источник
B. копирует значение
```

<details>
<summary>Ответ</summary>

B. `mov` копирует значение. Источник не очищается.

</details>

## Типовые ошибки

| Ошибка | Почему плохо |
|---|---|
| читать `add eax, ebx` как “просто сумма” | результат пишется в `eax` |
| думать, что `mov` очищает источник | `mov` копирует |
| путать регистр и память | `eax` и `[eax]` — разные вещи |
| забывать размер памяти | иногда нужен `byte`, `word`, `dword` |
| пытаться делать всё память-память | часто нужен промежуточный регистр |
