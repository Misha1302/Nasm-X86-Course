# День 19. Структуры: поля — это смещения

## Опора на материалы ВШЭ

`Slides2026-0xB.pdf`: структуры, поля, массив внутри структуры, linked list, alignment, padding.

## Зачем этот день

В C++ поле называется `r->a[i]`. В памяти имён нет. Есть базовый адрес, offset и scale.

## Главная мысль

Структура — непрерывный блок. Поле — фиксированное смещение. Padding нужен для выравнивания.

## Пример структуры

```cpp
struct rec {
    int i;      // offset 0
    int j;      // offset 4
    int a[3];   // offset 8
    rec* p;     // offset 20
};
```

Картинка:

```text
offset:  0      4      8      12     16     20
       +------+------+------+------+------+------+
field: |  i   |  j   | a[0] | a[1] | a[2] |  p   |
       +------+------+------+------+------+------+
```

Если `edx = r`, то:

| C++ access | ASM |
|---|---|
| `r->i` | `[edx]` |
| `r->j` | `[edx+4]` |
| `r->a[i]` | `[edx+4*eax+8]` |
| `r->p` | `[edx+20]` |

## Разбор

```asm
mov eax, [edx]                 ; eax = r->i
mov [edx + 4*eax + 8], ecx     ; r->a[i] = val
mov edx, [edx + 20]            ; r = r->p
```

## Padding

```cpp
struct S1 {
    char c;
    int x;
    char d;
};
```

В памяти может быть так:

```text
+---+---+---+---+---+---+---+---+---+---+---+---+
| c | p | p | p |       x       | d | p | p | p |
+---+---+---+---+---+---+---+---+---+---+---+---+
```

`p` — padding bytes.

Если переставить поля:

```cpp
struct S2 {
    int x;
    char c;
    char d;
};
```

структура может стать компактнее.

## Мини-челленджи

1. Объясни `[edx+4*eax+8]`.
2. Почему `sizeof(struct)` не всегда сумма размеров полей?
3. Что хранится в памяти: имена полей или байты?

<details>
<summary>Ответы / подсказки</summary>

1. База `edx`, массив `a` со смещения 8, элемент `eax`, размер int = 4.
2. Из-за padding/alignment.
3. Только байты. Имена — информация уровня исходного кода/отладки.

</details>
