## Главная мысль

В NASM:

```asm
x       ; адрес метки x
[x]     ; значение в памяти по адресу x
```

То есть `x` и `[x]` — это не одно и то же.

C++-аналогия:

```cpp
&x  // адрес переменной
x   // значение переменной
```

В NASM примерно:

```asm
x    ; как &x
[x]  ; как *(&x), то есть значение x
```

---

## Память как массив байтов

Можно представить память так:

```cpp
uint8_t memory[...];
```

Адрес — это номер байта.

```text
memory[address] = byte
```

Если мы храним `dword`, то он занимает 4 соседних байта.

---

## Секции NASM

В программе код и данные обычно лежат в разных секциях.

```asm
section .text   ; код
section .data   ; инициализированные данные
section .bss    ; зарезервированная память
section .rodata ; read-only данные, строки формата
```

### `.text`

Тут лежат инструкции:

```asm
section .text
global main

main:
    xor eax, eax
    ret
```

### `.data`

Тут лежат данные, у которых уже есть начальное значение:

```asm
section .data
    x dd 123
    y dd 456
    fmt db "%d", 10, 0
```

### `.bss`

Тут резервируют место без явной инициализации:

```asm
section .bss
    x resd 1      ; место под 1 dword
    arr resd 100  ; место под 100 dword
```

### `.rodata`

Часто там держат константы и строки формата. В учебных NASM-программах строки можно класть и в `.data`, но полезно знать идею: read-only данные должны быть отдельно.

---

## Директивы данных

| Директива | Смысл | Пример |
|---|---|---|
| `db` | define byte | `c db 10` |
| `dw` | define word, 2 байта | `s dw 1000` |
| `dd` | define dword, 4 байта | `x dd 123` |
| `dq` | define qword, 8 байт | `q dq 123456` |
| `resb` | reserve bytes | `buf resb 64` |
| `resw` | reserve words | `arr resw 10` |
| `resd` | reserve dwords | `arr resd 10` |
| `resq` | reserve qwords | `arr resq 10` |

Пример:

```asm
section .data
    a dd 10
    b dd 20

section .bss
    result resd 1
```

`a` и `b` уже имеют значения. `result` — просто зарезервированное место.

## Little-endian

IA-32 хранит многобайтовые числа в little-endian: младший байт лежит по младшему адресу.

```asm
section .data
x dd 0x12345678
```

В памяти:

```text
Address:   x      x+1    x+2    x+3
         +------+------+------+------+
Byte:    |  78  |  56  |  34  |  12  |
         +------+------+------+------+
```

Почему так?

Потому что младшая часть числа `0x78` лежит первой.

Важно: little-endian — это порядок байтов в памяти, а не порядок цифр в записи числа.

---

## Адрес против значения

Пусть есть:

```asm
section .data
    x dd 123
```

В памяти:

```text
address of x
    |
    v
+---------+
|   123   |
+---------+
```

Теперь команды:

```asm
mov eax, x      ; eax = address of x
mov eax, [x]    ; eax = value stored at x, то есть 123
mov [x], eax    ; store eax into memory at x
```

Таблица:

| NASM | Что происходит |
|---|---|
| `mov eax, x` | положить адрес `x` в `eax` |
| `mov eax, [x]` | прочитать значение из памяти по адресу `x` |
| `mov [x], eax` | записать значение `eax` в память по адресу `x` |

---

## Почему это важно для `scanf`

В C ты пишешь:

```c
scanf("%d", &x);
```

Почему `&x`? Потому что `scanf` должен знать, куда записать число.

В NASM это значит: передать адрес.

Правильно:

```asm
push x
push fmtIn
call scanf
add esp, 8
```

Неправильно:

```asm
push [x]
```

Почему неправильно?

`[x]` — это текущее значение в переменной. А `scanf` нужен адрес, куда записывать новое значение.

---

## Почему это важно для `printf`

В C:

```c
printf("%d", x);
```

Тут нужен не адрес, а значение.

В NASM:

```asm
push dword [x]
push fmtOut
call printf
add esp, 8
```

То есть:

| Функция | Что передаём |
|---|---|
| `scanf("%d", &x)` | адрес `x`, то есть `x` |
| `printf("%d", x)` | значение `x`, то есть `[x]` |

---

## Размер памяти: зачем `dword`

Иногда NASM понимает размер из регистра:

```asm
mov eax, [x]
```

Раз `eax` 32-битный, читается `dword`.

Но иногда размер не очевиден:

```asm
inc [x] ; NASM не знает: byte? word? dword?
```

Нужно сказать явно:

```asm
inc dword [x]
```

То же самое бывает с `push`:

```asm
push dword [x]
```

Так мы явно говорим: положить 4-байтовое значение.

---

## Мини-пример целиком

```asm
section .data
    fmtOut db "%d", 10, 0
    x dd 123

section .text
    extern printf
    global main

main:
    push dword [x] ; value 123
    push fmtOut    ; address of format string
    call printf
    add esp, 8

    xor eax, eax
    ret
```

Здесь:

```asm
push dword [x]
```

кладёт значение `123`.

А:

```asm
push fmtOut
```

кладёт адрес строки формата.

x resd 1

<summary>Ответ</summary>

`x`, потому что нужен адрес.

</details>

<summary>Ответ</summary>

## Типовые ошибки

| Ошибка | Почему плохо |
|---|---|
| `push [x]` для `scanf` | передаёшь значение вместо адреса |
| `push [fmt]` | строка формата нужна адресом, не первым dword строки |
| думать, что `[x]` — адрес | наоборот: это значение по адресу |
| забыть little-endian | неправильно читаешь байты в памяти |
| не указать `dword` там, где размер неясен | NASM может не собрать или понять не то |

После этого дня ты должен уверенно объяснять:
