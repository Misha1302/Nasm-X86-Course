# День 05. `x` и `[x]`: главная ловушка первых недель

## Опора на материалы ВШЭ

`Slides2026-02.pdf`, `Slides2026-04.pdf`: память, little-endian, секции `.data`, `.bss`, `.text`.

## Зачем этот день

Если перепутать `x` и `[x]`, сломается всё: `scanf`, переменные, массивы, структуры. Поэтому это один из самых важных дней.

## Главная мысль

`x` — адрес метки. `[x]` — значение в памяти по этому адресу.

## Секции

```text
section .text   -> код
section .data   -> инициализированные данные
section .bss    -> зарезервированная память
section .rodata -> read-only данные, строки формата
```

Объявления:

```asm
x dd 123      ; dword initialized with 123
buf resd 10   ; reserve 10 dwords
fmt db "%d", 0
```

## Little-endian

```asm
section .data
x dd 0x12345678
```

В памяти IA-32:

```text
Address:   x      x+1    x+2    x+3
         +------+------+------+------+
Byte:    |  78  |  56  |  34  |  12  |
         +------+------+------+------+
```

Младший байт лежит по младшему адресу.

## Адрес против значения

```asm
mov eax, x      ; eax = address of x
mov eax, [x]    ; eax = value stored at x
mov [x], eax    ; store eax into x
```

C++-аналогия:

| NASM | C++-смысл |
|---|---|
| `x` | примерно `&x` |
| `[x]` | примерно `*(&x)`, то есть значение `x` |

## Почему это важно для `scanf`

`scanf` должен записать число куда-то. Значит, ему нужен **адрес**.

```asm
push x      ; правильно: адрес x
push [x]    ; неправильно: текущее значение x
```

## Мини-челленджи

1. Разложи `0x01020304` по памяти IA-32.
2. Что делает `mov eax, x`?
3. Что делает `mov eax, [x]`?
4. Что нужно передавать в `scanf`: `x` или `[x]`?

<details>
<summary>Ответы / подсказки</summary>

1. `04 03 02 01`.
2. Кладёт адрес `x` в `eax`.
3. Загружает значение, лежащее по адресу `x`.
4. В `scanf` нужен адрес: `x`.

</details>

## Типовая ошибка

В NASM квадратные скобки почти всегда означают обращение к памяти. Видишь `[ ... ]` — думай: “идём по адресу”.
