# День 11. EFLAGS: лампочки после инструкции

## Опора на материалы ВШЭ

`Slides2026-04.pdf`, `Slides2026-06.pdf`: EFLAGS, влияние инструкций на флаги.

## Зачем этот день

У процессора нет типа `bool` для результата сравнения. Вместо этого после операций он включает флаги. Условные переходы потом смотрят на эти флаги.

## Главная мысль

Флаги — признаки результата. Они живут до следующей инструкции, которая их изменит.

```text
+----+----+----+----+----+----+
| OF | SF | ZF | AF | PF | CF |
+----+----+----+----+----+----+
  |    |    |    |    |    +-- unsigned carry/borrow
  |    |    |    |    +------- parity
  |    |    |    +------------ auxiliary carry
  |    |    +----------------- zero result
  |    +---------------------- sign bit
  +--------------------------- signed overflow
```

## Самые важные флаги

| Flag | Как думать |
|---|---|
| `ZF` | результат равен нулю |
| `SF` | старший бит результата равен 1 |
| `CF` | был перенос/заём для unsigned |
| `OF` | было signed overflow |
| `PF` | чётность младшего байта |
| `AF` | auxiliary carry, исторически для BCD |

## Примеры

```asm
xor eax, eax    ; eax = 0, ZF=1, CF=0, OF=0
add eax, ebx    ; обновляет арифметические флаги
sub eax, ebx    ; обновляет арифметические флаги
and eax, ebx    ; обновляет ZF/SF/PF, сбрасывает CF/OF
```

## Важная ловушка

```asm
cmp eax, ebx
add ecx, 1
je .equal
```

Так делать нельзя, если `je` должен относиться к `cmp`: `add` уже перезаписал флаги.

## Мини-челленджи

1. Что делает с флагами `xor eax, eax`?
2. Почему опасно вставлять `add` между `cmp` и `je`?
3. Чем `CF` отличается от `OF`?

<details>
<summary>Ответы / подсказки</summary>

1. Делает `eax=0`, ставит `ZF=1`, сбрасывает `CF/OF`.
2. `add` меняет флаги, и `je` будет смотреть уже не на результат `cmp`.
3. `CF` — unsigned carry/borrow, `OF` — signed overflow.

</details>
