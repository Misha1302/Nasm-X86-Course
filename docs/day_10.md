## За 30 секунд

- `and` оставляет нужные биты.
- `or` склеивает непересекающиеся поля.
- `xor` помогает инвертировать через маску.
- `not` инвертирует все биты.
- `shl` двигает биты влево.
- `shr` делит unsigned на степени двойки.
- `sar` сохраняет знак и нужен для branchless abs.

## Минимум после главы

Можно пока не заучивать:

- тонкости флагов после всех битовых операций;
- оптимальные формы кода, если базовая форма уже правильная.

::: warning C++-формулы здесь — про 32-битные биты
В C++-формулах этого дня мысленно работаем с `uint32_t`.

В NASM регистр хранит просто 32 бита. Но если ты переносишь формулу на C++, не делай signed-сдвиги в старший бит: используй unsigned-типы и явные маски.
:::

::: warning `%d` или `%u`?
Для обычных signed-задач печатаем через `%d`.

Для битовых задач, где ответ нужно понимать как unsigned 32-bit, используй `%u`:

```asm
fmtOut db "%u", 10, 0
```

Биты в `eax` те же самые. Меняется только то, как `printf` интерпретирует эти 32 бита.
:::

## Битовые команды в одной таблице

| Команда | Смысл | Где пригодится |
|---|---|---|
| `and` | оставить биты, где маска 1 | выделить поле, masked merge |
| `or` | поставить биты из двух частей | склеить поля |
| `xor` | 1 там, где биты различаются | branchless tricks |
| `not` | инвертировать все биты | `~mask` |
| `shl` | сдвиг влево | поставить байт на позицию |
| `shr` | логический сдвиг вправо | unsigned `/ 2^k` |
| `sar` | арифметический сдвиг вправо | signed mask |

---

## `shl`, `shr`, `sar`

```text
shl:  00010110 << 2 = 01011000
shr:  10010110 >> 2 = 00100101   ; слева нули
sar:  10010110 >> 2 = 11100101   ; слева знаковый бит
```

Главная разница:

| Команда | Слева появляются | Для чего обычно |
|---|---|---|
| `shr` | нули | unsigned |
| `sar` | копия знакового бита | signed |

Для переменного сдвига используется `cl`:

```asm
mov cl, [k]
shl eax, cl
```

## 01-7. Упаковка вектора

### Условие в одну строку

Есть четыре 8-битных числа `a`, `b`, `c`, `d`. Нужно собрать одно 32-битное число:

```text
31..24      23..16      15..8       7..0
+-----------+-----------+-----------+-----------+
|     d     |     c     |     b     |     a     |
+-----------+-----------+-----------+-----------+
```

### Формула

```cpp
uint32_t x =
    (a & 0xFFu) |
    ((b & 0xFFu) << 8) |
    ((c & 0xFFu) << 16) |
    ((d & 0xFFu) << 24);
```

### NASM

```asm
mov eax, [a]
and eax, 255

mov ecx, [b]
and ecx, 255
shl ecx, 8
or eax, ecx

mov ecx, [c]
and ecx, 255
shl ecx, 16
or eax, ecx

mov ecx, [d]
and ecx, 255
shl ecx, 24
or eax, ecx
```

### Ручные тесты

| `a` | `b` | `c` | `d` | expected hex |
|---:|---:|---:|---:|---|
| `1` | `2` | `3` | `4` | `0x04030201` |
| `13` | `12` | `11` | `10` | `0x0A0B0C0D` |
| `255` | `0` | `0` | `0` | `0x000000FF` |

### Типовая ошибка

| Ошибка | Почему плохо |
|---|---|
| забыть `and ..., 255` | если вход не гарантирован 8-bit, лишние биты попадут в ответ |
| смешать little-endian и формулу | упаковка значения и порядок байтов в памяти — разные вещи |

## 01-8. Masked merge

### Условие в одну строку

Для каждого бита выбрать:

```text
если c[i] = 1 -> взять бит из a
если c[i] = 0 -> взять бит из b
```

### Формула

```cpp
answer = (a & c) | (b & ~c);
```

### Картинка

```text
a      1 1 0 0
b      0 0 1 1
c      1 0 1 0
~c     0 1 0 1

a&c    1 0 0 0
b&~c   0 0 0 1
OR     1 0 0 1
```

### NASM

```asm
mov eax, [a]
and eax, [c]

mov ecx, [c]
not ecx
and ecx, [b]

or eax, ecx
```

### Ручные тесты

| `c` | expected |
|---|---|
| `00000000h` | `b` |
| `FFFFFFFFh` | `a` |
| `FF00FF00h` | верхние байты из `a`, остальные из `b` по маске |

### Типовая ошибка

| Ошибка | Почему плохо |
|---|---|
| написать `(a & ~c) | (b & c)` | выбираешь наоборот |
| забыть `not c` для второй части | биты будут взяты не из тех позиций |

## 01-9. Модуль числа без переходов

### Условие в одну строку

Найти `abs(x)`, но нельзя использовать условные переходы и `cmov`.

### Идея маски

```text
x >= 0 -> mask = 00000000h
x < 0  -> mask = FFFFFFFFh
```

Получаем через:

```asm
mov edx, eax
sar edx, 31
```

### Формула

```text
answer = (x ^ mask) - mask
```

### NASM

```asm
mov eax, [x]
mov edx, eax
sar edx, 31
xor eax, edx
sub eax, edx
```

### Почему работает

| `x` | `mask` | `(x ^ mask) - mask` |
|---:|---:|---|
| `123` | `0` | `123` |
| `-123` | `-1` | `~x + 1 = 123` |

Про `INT_MIN`: математически `abs(INT_MIN)` не помещается в signed 32-bit. Если в условии сказано, что ответ помещается в 32 бита, этот случай исключён.

### Ручные тесты

| `x` | expected |
|---:|---:|
| `0` | `0` |
| `1` | `1` |
| `-1` | `1` |
| `123` | `123` |
| `-123` | `123` |

### Типовая ошибка

| Ошибка | Почему плохо |
|---|---|
| `shr edx, 31` | для отрицательного числа получится `1`, а нужна маска `FFFFFFFFh` |
| использовать `jcc` | нарушает запрет задачи |

## 01-10. Год на Пандоре

### Условие в одну строку

В году 14 месяцев. Нечётные месяцы — 41 день, чётные — 42 дня. Даны `month`, `day`, оба с 1. Найти номер дня в году.

### Формула

```cpp
before = month - 1;
answer = before * 41 + before / 2 + day;
```

Почему `before / 2`?

Каждый предыдущий месяц даёт минимум 41 день. Каждый второй предыдущий месяц был чётным и даёт ещё +1 день.

### NASM

```asm
mov eax, [month]
sub eax, 1        ; eax = before

mov ecx, eax      ; ecx = before
imul eax, 41      ; eax = before * 41
shr ecx, 1        ; ecx = before / 2, before >= 0
add eax, ecx
add eax, [day]
```

### Ручные тесты

| `month` | `day` | expected |
|---:|---:|---:|
| `1` | `1` | `1` |
| `2` | `1` | `42` |
| `3` | `1` | `84` |
| `4` | `1` | `125` |

### Типовая ошибка

| Ошибка | Почему плохо |
|---|---|
| забыть `month - 1` | будет сдвиг на один месяц |
| писать цикл/if вместо формулы | сложнее и больше шанс ошибиться |
| использовать `sar` без причины | `before >= 0`, достаточно `shr` |

## Практика

### A. Trace

Что получится?

```asm
mov eax, 0x0000000D
mov ecx, 0x0000000C
shl ecx, 8
or eax, ecx
```

<details>
<summary>Ответ</summary>

`eax = 0x00000C0D`.

</details>

### B. Заполни пропуски

```asm
; branchless abs
mov eax, [x]
mov edx, eax
___ edx, 31
xor eax, edx
sub eax, ___
```

<details>
<summary>Ответ</summary>

```asm
mov eax, [x]
mov edx, eax
sar edx, 31
xor eax, edx
sub eax, edx
```

</details>

### C. Напиши сам

Напиши фрагмент для masked merge:

```cpp
answer = (a & c) | (b & ~c);
```

<details>
<summary>Ответ</summary>

```asm
mov eax, [a]
and eax, [c]

mov ecx, [c]
not ecx
and ecx, [b]

or eax, ecx
```

</details>

### D. Найди баг

```asm
mov eax, [x]
mov edx, eax
shr edx, 31
xor eax, edx
sub eax, edx
```

Почему это ломает `abs` для отрицательных?

<details>
<summary>Ответ</summary>

`shr` даёт `0` или `1`, а нужна маска `0` или `FFFFFFFFh`. Нужно `sar edx, 31`.

</details>
