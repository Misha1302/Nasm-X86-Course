## За 30 секунд

- `and` оставляет нужные биты.
- `or` склеивает независимые части.
- `xor` меняет биты там, где маска равна `1`.
- `not` инвертирует все 32 бита.
- `shl` двигает биты влево.
- `shr` — unsigned-сдвиг вправо.
- `sar` — signed-сдвиг вправо, нужен для маски знака.
- Если условные переходы запрещены, обычно ищем формулу, маску или одинаковую операцию над всеми битами.

В C++-формулах этой главы мысленно работаем с `uint32_t`.

В NASM регистр хранит просто 32 бита. Но если переносишь формулу на C++, не делай signed-сдвиги в старший бит: используй unsigned-типы и явные маски.

Для signed-чисел печатаем через `%d`.

Для unsigned 32-bit — через `%u`.

Для hex-задач вроде masked merge — через `%x`:

```asm

```

## Битовые команды в одной таблице

| Команда | Смысл | Где пригодится |
|---|---|---|
| `and` | оставить биты, где маска 1 | masked merge, выделение поля |
| `or` | объединить непересекающиеся части | сборка результата |
| `xor` | перевернуть биты по маске | branchless `abs`, битовые системы |
| `not` | заменить `0` на `1`, `1` на `0` | `~mask` |
| `shl` | сдвиг влево | умножение на `2^k`, упаковка |
| `shr` | сдвиг вправо с нулями | unsigned `/ 2^k` |
| `sar` | сдвиг вправо со знаком | `0/-1` mask |

## Главный branchless-паттерн

Если есть маска:

```text
mask = 00000000h  -> выбрать b
mask = FFFFFFFFh  -> выбрать a
```

то выбор без `if`:

```text
answer = (a & mask) | (b & ~mask)
```

NASM-shape:

```asm
; eax = a
; edx = b
; ecx = mask: 0 или -1

and eax, ecx
not ecx
and edx, ecx
or eax, edx
```

Главная ошибка: использовать маску `0/1`. Для выбора всего числа нужна маска `0/FFFFFFFFh`.

Подробнее: [Branchless-маски](/patterns/branchless).

## `ceil(a / b)` для неотрицательных чисел

В задачах часто нужно округлить деление вверх.

```text
ceil(a / b) = (a + b - 1) / b
```

Для `a >= 0`, `b > 0`:

```asm
mov eax, [a]
add eax, [b]
dec eax
xor edx, edx
div dword [b]
; eax = ceil(a / b)
```

Где пригодится:

- количество коробок в 01-14;
- количество задержанных коробок с номерами `0, 3, 6, ...`.

## 01-4. Книжки

### Смысл

В начале 2011 года есть `X` книг. Каждый год сначала покупают `N`, потом теряют `M`.

К началу года `Y` прошло:

```text
years = Y - 2011
```

Ответ:

```text
answer = X + years * (N - M)
```

### NASM-shape

```asm
mov eax, [Y]
sub eax, 2011        ; years

mov ecx, [N]
sub ecx, [M]         ; delta

imul eax, ecx
add eax, [X]
```

Здесь нет условий. Запрет `jcc` не мешает.

Подробно: [01-4 Книжки](/tasks/spring-01/01-04-books).

## 01-8. Masked merge

### Смысл

Для каждого бита выбрать:

```text
если c[i] = 1 -> взять a[i]
если c[i] = 0 -> взять b[i]
```

Формула:

```text
d = (a & c) | (b & ~c)
```

NASM:

```asm
mov eax, [a]
and eax, [c]

mov ecx, [c]
not ecx
and ecx, [b]

or eax, ecx
```

Подробно: [01-8 Masked merge](/tasks/spring-01/01-08-masked-merge).

## 01-14. Огород

### Смысл

Сначала считаем коробки:

```text
items = N * M * K
boxes = ceil(items / D)
```

Ночью проходят все:

```text
night = boxes
```

Днём задерживаются коробки с номерами `0, 3, 6, ...`.

Среди `boxes` коробок таких номеров:

```text
blocked = ceil(boxes / 3)
day = boxes - blocked
```

Остаётся выбрать:

```text
если X < 6 -> night
иначе     -> day
```

Но условные переходы запрещены, значит нужен branchless-выбор по маске.

Подробно: [01-14 Огород](/tasks/spring-01/01-14-garden).

## 01-15. Площадь треугольника

### Смысл

Площадь через determinant:

```text
det = (x2 - x1) * (y3 - y1) - (x3 - x1) * (y2 - y1)
area = abs(det) / 2
```

Так как нужен один знак после запятой, float не обязателен.

Если `abs(det)` чётный:

```text
answer = abs(det)/2 .0
```

Если нечётный:

```text
answer = abs(det)/2 .5
```

NASM-идея:

```text
absDet = branchless_abs(det)
integerPart = absDet / 2
fractionDigit = (absDet % 2) * 5
```

Подробно: [01-15 Площадь](/tasks/spring-01/01-15-triangle-area).

## 01-16. Система уравнений

### Смысл

Дана система:

```text
(a11 & x) ^ (a12 & y) = b1
(a21 & x) ^ (a22 & y) = b2
```

Каждый бит независим от остальных. Значит, можно решать 32 маленькие системы одновременно через 32-битные маски.

Для одного бита есть четыре кандидата:

```text
x=0 y=0
x=1 y=0
x=0 y=1
x=1 y=1
```

Для каждого кандидата строим маску тех битов, где он подходит. Потом выбираем одно решение для каждого бита.

Подробно: [01-16 Система](/tasks/spring-01/01-16-bit-system).

## Практика

### A. Trace

Что лежит в `eax` после кода?

```asm
mov eax, 0b1100
mov ecx, 0b1010
and eax, ecx
```

### B. Заполни пропуски

```asm
; d = (a & c) | (b & ~c)
mov eax, [a]
___ eax, [c]

mov ecx, [c]
___ ecx
and ecx, [b]

___ eax, ecx
```

### C. Напиши сам

Дано `items` и `D`. Напиши `ceil(items / D)` для unsigned-чисел.

### D. Найди баг

```asm
; хотели выбрать a при mask=1, b при mask=0
and eax, ecx
not ecx
and edx, ecx
or eax, edx
```

Что не так?

Ответ: для выбора всего числа `mask` должна быть `0` или `FFFFFFFFh`, а не `0` или `1`.
