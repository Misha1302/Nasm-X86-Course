## Главная мысль

Маска — это способ сделать много маленьких выборов без `if`.

Сдвиги позволяют двигать биты на нужные позиции.

```text
and / or / xor / not  -> выбираем и склеиваем биты
shl / shr / sar       -> двигаем биты
```

# 1. Битовые операции

## `and`

`and` оставляет 1 только там, где в обоих операндах 1.

```text
  11001010
& 11110000
-----------
  11000000
```

NASM:

```asm
and eax, ebx
```

C++:

```cpp
eax = eax & ebx;
```

Типичное применение: оставить только нужные биты.

---

## `or`

`or` ставит 1 там, где хотя бы в одном операнде 1.

```text
  11001010
| 00001111
-----------
  11001111
```

NASM:

```asm
or eax, ebx
```

C++:

```cpp
eax = eax | ebx;
```

Типичное применение: склеить непересекающиеся битовые части.

---

## `xor`

`xor` ставит 1 там, где биты различаются.

```text
  11001010
^ 11110000
-----------
  00111010
```

NASM:

```asm
xor eax, ebx
```

C++:

```cpp
eax = eax ^ ebx;
```

Частый трюк:

```asm
xor eax, eax
```

Это делает `eax = 0`, потому что любое число xor само с собой даёт 0.

---

## `not`

`not` инвертирует все биты.

```text
~10110000 = 01001111
```

NASM:

```asm
not eax
```

C++:

```cpp
eax = ~eax;
```

Важно: инвертируются все 32 бита, если работаешь с `eax`.

---

# 2. Сдвиги

## `shl` / `sal`

Сдвиг влево.

```text
00010110 << 2 = 01011000
```

NASM:

```asm
shl eax, 8
```

C++:

```cpp
eax <<= 8;
```

Если нет переполнения, сдвиг влево на `k` похож на умножение на `2^k`.

```text
x << 8 = x * 256
```

---

## `shr`

Логический сдвиг вправо. Слева добавляются нули.

```text
10010110 shr 2 = 00100101
```

Это хорошо для unsigned-чисел.

---

## `sar`

Арифметический сдвиг вправо. Слева копируется знаковый бит.

```text
10010110 sar 2 = 11100101
```

Это важно для signed-чисел и особенно для задачи 01-9.

`shr` и `sar` отличаются только для чисел со старшим битом 1.

---

## Переменный сдвиг через `cl`

Если величина сдвига не константа, x86 использует `cl`.

```asm
mov cl, [k]
shl eax, cl
```

`cl` — младший байт регистра `ecx`.

# 3. 01-7. Упаковка вектора

## Условие простыми словами

Есть четыре 8-битных числа:

```text
a, b, c, d
```

Нужно собрать одно 32-битное число так, чтобы:

```text
младшие 8 бит      = a
следующие 8 бит    = b
следующие 8 бит    = c
старшие 8 бит      = d
```

Картинка:

```text
31..24      23..16      15..8       7..0
+-----------+-----------+-----------+-----------+
|     d     |     c     |     b     |     a     |
+-----------+-----------+-----------+-----------+
```

## Формула

```cpp
x = a | (b << 8) | (c << 16) | (d << 24);
```

Почему `or`, а не `add`?

Потому что поля не пересекаются. Мы просто ставим каждую часть на своё место и склеиваем биты.

## Пример

```text
a = 1
b = 2
c = 3
d = 4
```

Тогда:

```text
x = 0x04030201 = 67305985
```

Обрати внимание: в записи числа слева виден `d`, справа `a`. Это не противоречит little-endian. Здесь мы говорим о значении числа, а не о порядке байтов в памяти.

## NASM-идея

```asm
mov eax, [a]

mov ebx, [b]
shl ebx, 8
or eax, ebx

mov ebx, [c]
shl ebx, 16
or eax, ebx

mov ebx, [d]
shl ebx, 24
or eax, ebx
```

Если `a`, `b`, `c`, `d` гарантированно 8-битные, дополнительная маска `and ..., 255` не нужна. Если не гарантированно — лучше маскировать.

## Более осторожный вариант

```asm
mov eax, [a]
and eax, 255

mov ebx, [b]
and ebx, 255
shl ebx, 8
or eax, ebx

mov ebx, [c]
and ebx, 255
shl ebx, 16
or eax, ebx

mov ebx, [d]
and ebx, 255
shl ebx, 24
or eax, ebx
```

# 4. 01-8. Masked merge

## Условие простыми словами

Для каждого бита нужно выбрать:

```text
если c[i] = 1 -> взять бит из a
если c[i] = 0 -> взять бит из b
```

То есть маска `c` говорит, откуда брать каждый бит.

## Формула

```cpp
d = (a & c) | (b & ~c);
```

Почему работает:

```text
a & c   -> оставляет биты a там, где c=1
b & ~c  -> оставляет биты b там, где c=0
or      -> склеивает две части
```

## Маленькая картинка

```text
a      1 1 0 0
b      0 0 1 1
c      1 0 1 0
~c     0 1 0 1

a&c    1 0 0 0
b&~c   0 0 0 1
OR     1 0 0 1
```

## NASM

```asm
mov eax, [a]
and eax, [c]

mov ebx, [c]
not ebx
and ebx, [b]

or eax, ebx
```

Результат в `eax`.

## Проверки

Если:

```text
c = 0
```

то:

```text
a & c = 0
b & ~c = b
answer = b
```

Если:

```text
c = 0xFFFFFFFF
```

то:

```text
a & c = a
b & ~c = 0
answer = a
```

Это отличные крайние тесты.

# 5. 01-9. Модуль числа без переходов

## Условие простыми словами

Нужно вычислить:

```cpp
abs(x)
```

Но нельзя использовать:

- условные переходы `jcc`;
- условную передачу `cmov`;
- любые ветвления по условию.

Обычный C++-вариант нельзя:

```cpp
if (x < 0) x = -x;
```

Нужно сделать через биты.

## Идея маски знака

Для 32-битного signed `x`:

```cpp
mask = x >> 31;
```

В NASM:

```asm
mov edx, eax
sar edx, 31
```

Если `x >= 0`:

```text
mask = 00000000h
```

Если `x < 0`:

```text
mask = FFFFFFFFh = -1
```

Почему нужен `sar`, а не `shr`?

Потому что `sar` копирует знаковый бит. Для отрицательного числа сверху должны появиться единицы.

## Формула

```cpp
answer = (x ^ mask) - mask;
```

## Случай 1: `x >= 0`

```text
mask = 0
answer = (x ^ 0) - 0 = x
```

## Случай 2: `x < 0`

```text
mask = -1 = FFFFFFFFh
answer = (x ^ -1) - (-1)
       = ~x + 1
       = -x
```

Это ровно two's complement.

## NASM

```asm
mov eax, [x]
mov edx, eax
sar edx, 31
xor eax, edx
sub eax, edx
```

Результат в `eax`.

## Проверки

```text
x = 123  -> 123
x = -123 -> 123
x = 0    -> 0
```

Про `INT_MIN`: математически `abs(INT_MIN)` не помещается в signed 32-bit. В условии обычно сказано, что ответ помещается в 32 бита, значит этот опасный случай исключён.

# 6. 01-10. Год на Пандоре

## Условие простыми словами

В году 14 месяцев.

Нечётные месяцы имеют 41 день.

Чётные месяцы имеют 42 дня.

Даны:

```text
month, day
```

оба с 1.

Нужно вывести номер дня в году.

## Не надо делать `if`

Можно было бы думать:

```cpp
for months before current:
    if month is odd add 41
    else add 42
```

Но для такой задачи лучше вывести формулу.

Количество предыдущих месяцев:

```cpp
before = month - 1;
```

Каждый предыдущий месяц даёт минимум 41 день:

```cpp
before * 41
```

Каждый второй предыдущий месяц был чётным и даёт ещё +1 день:

```cpp
before / 2
```

И добавляем текущий день:

```cpp
answer = before * 41 + before / 2 + day;
```

## Проверка формулы

### Месяц 1, день 1

```text
before = 0
answer = 0*41 + 0/2 + 1 = 1
```

### Месяц 2, день 1

Перед ним только месяц 1 длиной 41:

```text
before = 1
answer = 1*41 + 1/2 + 1 = 41 + 0 + 1 = 42
```

### Месяц 3, день 1

Перед ним месяцы 1 и 2:

```text
41 + 42 = 83
answer = 84
```

Формула:

```text
before = 2
answer = 2*41 + 2/2 + 1 = 82 + 1 + 1 = 84
```

## NASM

```asm
mov eax, [month]
sub eax, 1        ; eax = before

mov ebx, eax      ; ebx = before
imul eax, 41      ; eax = before * 41
shr ebx, 1        ; ebx = before / 2, before >= 0
add eax, ebx
add eax, [day]
```

Почему `shr`, а не `sar`?

`before` неотрицательный, поэтому логический сдвиг вправо нормально делит на 2.

# 7. Мини-челленджи

### 1. Упаковка `13 12 11 10`

Какое число получится в hex?

<details>
<summary>Ответ</summary>

```text
a=13=0D
b=12=0C
c=11=0B
d=10=0A

answer = 0x0A0B0C0D
```

</details>

### 2. Masked merge при `c = 0`

Что получится?

<details>
<summary>Ответ</summary>

Получится `b`, потому что все биты берутся из `b`.

</details>

### 3. Masked merge при `c = FFFFFFFFh`

Что получится?

<details>
<summary>Ответ</summary>

Получится `a`, потому что все биты берутся из `a`.

</details>

### 4. Branchless abs для `-123`

Объясни словами.

<details>
<summary>Ответ</summary>

Для отрицательного числа `sar edx,31` даёт `edx=-1`. Потом `x ^ -1` превращает `x` в `~x`, а `sub eax, edx` фактически добавляет 1. Получается `~x + 1 = -x`.

</details>

### 5. Пандора: месяц 4, день 1

Какой номер дня?

<details>
<summary>Ответ</summary>

Перед месяцем 4 идут месяцы 1, 2, 3:

```text
41 + 42 + 41 = 124
```

День 1 месяца 4:

```text
125
```

Формула:

```text
before = 3
3*41 + 3/2 + 1 = 123 + 1 + 1 = 125
```

</details>

# 8. Типовые ошибки

| Ошибка | Почему плохо |
|---|---|
| в 01-7 забыть один сдвиг | байт попадёт не в своё поле |
| в 01-7 смешать little-endian и формулу | упаковка значения и порядок байтов в памяти — разные вещи |
| в 01-8 перепутать `c` и `~c` | выберешь биты из неправильного числа |
| в 01-9 использовать `shr` вместо `sar` | для отрицательных чисел маска будет неправильной |
| в 01-9 случайно использовать `jcc` | нарушишь запрет задачи |
| в 01-10 писать цикл/if вместо формулы | сложнее, больше шанс ошибиться |
| в 01-10 забыть, что month/day с 1 | будет сдвиг на один день/месяц |
