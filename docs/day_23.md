# День 23. x87: вещественные числа как стек

## Опора на материалы ВШЭ

`Slides2026-0x10.pdf`: x87 FPU, `FINIT`, регистры `st0..st7`, control/status/tag word, `fld`, `fst`, `fstp`, `faddp`, `fsubp`, `fmulp`, `fdivp`, печать через `printf("%f")`, связь со стековой моделью и обратной польской записью.

## Зачем этот день

До сих пор мы работали в основном с целыми числами в регистрах:

```text
eax, ebx, ecx, edx
```

Но вещественные числа исторически обрабатывались отдельным блоком — x87 FPU.

x87 не похож на обычные регистры общего назначения. Он работает как стек:

```text
положили число;
положили второе;
операция взяла верхние элементы;
результат остался на вершине.
```

Сначала это непривычно. Но если думать как об обратной польской записи, становится нормально.

---

## Главная мысль

x87 — стековая машина для floating point.

```text
fld   -> push floating value
faddp -> add top values and pop
fstp  -> store and pop
```

Для `printf("%f")` нужно передавать `double`, то есть `qword`, даже если исходное число было `float`.

---

## 1. x87 stack

x87 имеет стек из 8 регистров:

```text
+-----------+
|   st(7)   |
+-----------+
|   st(6)   |
+-----------+
|   st(5)   |
+-----------+
|   st(4)   |
+-----------+
|   st(3)   |
+-----------+
|   st(2)   |
+-----------+
|   st(1)   |
+-----------+
|   st(0)   | <- top
+-----------+
```

`st(0)` — вершина стека.

После `fld x` значение оказывается в `st(0)`.

После второго `fld y`:

```text
st(0) = y
st(1) = x
```

---

## 2. `finit`

```asm
finit
```

Инициализирует x87 FPU: приводит его состояние к нормальному начальному виду.

В маленьких учебных программах `finit` часто ставят в начале работы с x87, чтобы не зависеть от старого состояния.

---

## 3. Загрузка и сохранение

### `fld`

```asm
fld dword [x]
```

Читает floating point значение из памяти и кладёт на вершину x87-стека.

```text
before:
empty

after fld [x]:
st(0) = x
```

Можно загружать разные размеры:

```asm
fld dword [floatValue]   ; 32-bit float
fld qword [doubleValue]  ; 64-bit double
```

### `fst`

```asm
fst dword [x]
```

Сохраняет `st(0)` в память, но не снимает его со стека.

### `fstp`

```asm
fstp dword [x]
```

Сохраняет `st(0)` в память и делает pop.

То есть:

```text
fst  -> store
fstp -> store + pop
```

---

## 4. Сложение через x87

Пусть есть:

```asm
section .data
    x dd 1.5
    y dd 2.25
    z dd 0.0
```

Код:

```asm
finit
fld dword [x]
fld dword [y]
faddp
fstp dword [z]
```

Разберём по шагам.

После:

```asm
fld dword [x]
```

```text
st(0) = x
```

После:

```asm
fld dword [y]
```

```text
st(0) = y
st(1) = x
```

После:

```asm
faddp
```

x87 складывает верхние элементы и снимает один:

```text
st(0) = x + y
```

После:

```asm
fstp dword [z]
```

```text
z = x + y
stack is empty again
```

---

## 5. Арифметические команды

| Instruction | Meaning |
|---|---|
| `faddp` | сложить верхние значения и pop |
| `fsubp` | вычесть и pop |
| `fmulp` | умножить и pop |
| `fdivp` | разделить и pop |

В учебной модели удобно думать так:

```text
fld a
fld b
operation
```

похоже на:

```text
a b operation
```

в обратной польской записи.

---

## 6. Порядок для `fsubp` и `fdivp`

Сложение и умножение коммутативны: порядок не так страшен.

А вот вычитание и деление требуют внимания. Безопаснее писать явную форму, где видно оба операнда.

### Шаблон `a - b`

```asm
fld dword [a]
fld dword [b]
fsubp st1, st0      ; st1 = st1 - st0, pop
```

После этого:

```text
st(0) = a - b
```

### Шаблон `a / b`

```asm
fld dword [a]
fld dword [b]
fdivp st1, st0      ; st1 = st1 / st0, pop
```

После этого:

```text
st(0) = a / b
```

Практический тест:

```text
a = 10
b = 2

a - b = 8
a / b = 5
```

Если получил `-8` или `0.2`, порядок перепутан.

## 7. Печать через `printf("%f")`

В C variadic-функции вроде `printf` получают `float` как `double`.

То есть:

```c
printf("%f", value);
```

ожидает 8 байт `double`.

В NASM это значит: перед `printf` нужно положить на стек `qword`.

Пример:

```asm
section .data
    fmt db "%f", 10, 0
    x dd 1.5
    y dd 2.25

section .text
    extern printf
    global main

main:
    finit
    fld dword [x]
    fld dword [y]
    faddp

    sub esp, 8
    fstp qword [esp]
    push fmt
    call printf
    add esp, 12

    xor eax, eax
    ret
```

Разбор:

```asm
sub esp, 8
fstp qword [esp]
```

Выделили 8 байт на стеке и сохранили туда `double`.

```asm
push fmt
call printf
add esp, 12
```

Аргументы `printf`:

```text
format pointer -> 4 bytes
floating value -> 8 bytes
итого 12 bytes
```

## 8. Типовая ошибка с `%f`

Плохо:

```asm
fstp dword [esp+4]
```

если дальше:

```asm
call printf
```

с форматом:

```c
"%f"
```

Почему плохо?

`%f` ждёт `double` — 8 байт. А `dword` — только 4 байта `float`. `printf` прочитает не те байты и напечатает мусор.

Правильно:

```asm
fstp qword [somewhere]
```

## 9. x87 status/control/tag word на уровне идеи

В x87 есть не только регистры `st(0)..st(7)`, но и служебные слова состояния.

| Word | Идея |
|---|---|
| control word | режимы округления, маски исключений |
| status word | состояние FPU, исключения, condition flags |
| tag word | какие регистры стека заняты/пусты |

Для задач обычно не нужно вручную ими управлять. Но важно знать: x87 — не просто “8 double-регистров”, это отдельная подсистема со своим состоянием.

## 10. Связь с обратной польской записью

Выражение:

```text
(a + b) * c
```

В обратной польской записи:

```text
a b + c *
```

x87-идея:

```asm
fld a
fld b
faddp
fld c
fmulp
```

То есть сначала кладём операнды, потом применяем операцию к верхушке стека.

Это полезная модель для сложных выражений.

## 11. Мини-челленджи

#### 1. Нарисуй x87 stack

После:

```asm
fld dword [x]
fld dword [y]
```

что в `st(0)` и `st(1)`?

<details>
<summary>Ответ</summary>

```text
st(0) = y
st(1) = x
```

</details>

#### 2. Что делает `fstp`?

<details>
<summary>Ответ</summary>

Сохраняет `st(0)` в память и снимает значение со x87-стека.

</details>

#### 3. Почему `%f` ждёт `qword`?

<details>
<summary>Ответ</summary>

Потому что в variadic C-функциях `float` продвигается до `double`, а `printf("%f")` ожидает 64-битный double.

</details>

#### 4. Как посчитать `(a+b)*c`?

<details>
<summary>Один вариант</summary>

```asm
fld dword [a]
fld dword [b]
faddp
fld dword [c]
fmulp
```

</details>

#### 5. Почему для `fsubp/fdivp` надо быть внимательнее?

<details>
<summary>Ответ</summary>

Потому что вычитание и деление не коммутативны. Если перепутать порядок операндов, получится другой результат.

</details>

## 12. Типовые ошибки

| Ошибка | Почему плохо |
|---|---|
| думать, что x87 использует `eax/ebx` | x87 имеет свой стек `st(0)..st(7)` |
| забыть `fstp` и оставить лишние значения на стеке | x87-стек может переполниться или запутать следующие операции |
| передать `dword` в `printf("%f")` | `%f` ждёт `double`, то есть `qword` |
| не выделить место на стеке под `qword` | запись испортит чужие данные |
| перепутать порядок `fsubp/fdivp` | знак/результат будет неправильным |
| не проверить простым примером | x87-ошибки часто выглядят как “странное число” |
