# День 22. Вещественные числа: почему `0.1` не точное

## Опора на материалы ВШЭ

`Slides2026-0xF.pdf`, `Slides2026-15.pdf`: двоичные дроби, floating point, normalized/denormalized, NaN, Inf.

## Зачем этот день

С целыми числами ещё можно думать “битовый вектор + интерпретация”. С вещественными хитрее: многие десятичные дроби бесконечны в двоичной записи.

## Главная мысль

Floating point — приближённое представление: знак, порядок, мантисса. Есть нормальные числа, денормалы, нули, бесконечности и NaN.

## Двоичные дроби

```text
5.75 decimal = 101.11 binary

5    = 101
0.75 = 1/2 + 1/4 = .11
```

А вот:

```text
1/10 = 0.0001100110011... in binary
```

Поэтому `0.1` не представляется точно конечным количеством битов.

## Layout float

```text
float, 32 bits:

31     30          23 22                    0
+--------+------------+----------------------+
| sign   | exponent   | fraction             |
+--------+------------+----------------------+
```

Идея значения:

```text
(-1)^sign * mantissa * 2^exponent
```

## Special values

| exponent | fraction | Meaning |
|---|---|---|
| all zero | zero | +0 / -0 |
| all zero | non-zero | denormalized |
| normal | any | normalized |
| all ones | zero | infinity |
| all ones | non-zero | NaN |

## Мини-челленджи

1. Представь `5.75` в двоичном виде.
2. Почему `1/10` бесконечна в binary?
3. Что означает exponent all ones и fraction zero?
4. Почему NaN — не “очень большое число”?

<details>
<summary>Ответы / подсказки</summary>

1. `101.11₂`.
2. Потому что знаменатель 10 содержит множитель 5, а конечные двоичные дроби имеют знаменатель степени 2.
3. Infinity.
4. NaN означает “не число”, результат некорректной/неопределённой операции.

</details>
