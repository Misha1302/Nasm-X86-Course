# Паттерн: связность через массивы

## Когда нужен

Когда `N` и `M` до `100000`, а задача требует много перестановок сегментов.

Физически двигать элементы массива после каждой операции нельзя: получится слишком медленно.

## Идея

Храним не сам порядок как массив значений, а связи:

```text
next[value] = следующий элемент
prev[value] = предыдущий элемент
head        = первый элемент
```

Перенос сегмента `[b..c]` в начало — это несколько переподключений ссылок.

## Shape операции

```text
before = prev[b]
after  = next[c]

отрезать [b..c]
приклеить [b..c] перед head
head = b
```

## Почему это быстро

Каждая операция меняет O(1) ссылок. Финальный вывод идёт одним проходом от `head`.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| реально сдвигать массив | O(NM), не пройдёт |
| забыть обновить `head` | вывод начнётся не оттуда |
| не обработать случай `b == head` | сегмент уже в начале |
| сломать `prev/next` симметрию | список становится несвязным |
