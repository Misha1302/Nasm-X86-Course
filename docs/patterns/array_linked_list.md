# Паттерн: связность через массивы

## Когда нужен

Когда в задаче есть много операций над порядком элементов:

- перенести сегмент в начало;
- удалить/вставить отрезок;
- `N` и `M` до `100000`;
- наивный сдвиг массива был бы слишком медленным.

Если после каждой операции реально двигать элементы массива, получится `O(NM)`. Для `100000 * 100000` это невозможно.

## Главная идея

Не двигаем элементы. Храним связи между значениями:

```text
next[value] = следующий элемент
prev[value] = предыдущий элемент
head        = первый элемент
```

Это linked list, но узлы лежат не в куче, а в массивах.

## Начальное состояние

Для последовательности:

```text
1 2 3 4 5 6 7
```

связи такие:

```text
next[1] = 2
next[2] = 3
...
next[7] = 0

prev[1] = 0
prev[2] = 1
...
prev[7] = 6

head = 1
```

`0` удобно использовать как “нет элемента”.

## Операция: перенести `[b..c]` в начало

Дано, что `b` встречается не позже `c`.

Нужно:

```text
before = prev[b]
after  = next[c]
```

### Шаг 1. Если `b == head`

Сегмент уже начинается в начале. Ничего переносить не надо.

### Шаг 2. Отрезать сегмент

```text
next[before] = after
if after != 0:
    prev[after] = before
```

Теперь `[b..c]` временно отдельно.

### Шаг 3. Приклеить перед старым head

```text
prev[head] = c
next[c] = head
prev[b] = 0
head = b
```

Сегмент стал началом списка.

## Почему это быстро

Каждая операция меняет только несколько связей:

```text
prev[b], next[c], next[before], prev[after], prev[head], head
```

Это `O(1)` на операцию.

Финальный вывод — один проход:

```text
cur = head
while cur != 0:
    print cur
    cur = next[cur]
```

## NASM-адресация

Если `next` и `prev` — массивы `dword`:

```asm
section .bss
    next resd 100001
    prev resd 100001
```

то:

```asm
mov eax, [next + 4*ecx]     ; eax = next[ecx]
mov [prev + 4*ecx], eax     ; prev[ecx] = eax
```

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| реально сдвигать массив | `O(NM)`, не пройдёт |
| забыть случай `b == head` | можно сломать начало списка |
| обновить `next`, но не `prev` | список станет несогласованным |
| забыть `after == 0` | запись в `prev[0]` может быть логически неверной |
| выводить от `1`, а не от `head` | порядок после операций потерян |

## Мини-практика

Исходно:

```text
1 2 3 4 5 6 7
```

Перенести `[4..6]` в начало:

```text
before = 3
after = 7
result = 4 5 6 1 2 3 7
```

Проверь, какие `next/prev` изменились.
