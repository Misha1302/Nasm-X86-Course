# Паттерн: libc и 16-byte stack alignment

## Когда нужен

В Spring-04 часто написано:

```text
Запрещается использовать io.inc.
При вызове всех библиотечных функций стек должен быть выровнен по 16 байт.
```

Это относится к `printf`, `scanf`, `fopen`, `fscanf`, `fprintf`, `strstr`, `strcmp` и похожим функциям.

## За 30 секунд

Обычного “после `call` сделал `add esp, ...`” уже мало. Перед библиотечным `call` нужно ещё контролировать кратность `esp`.

## Практический подход курса

Для Spring-04 лучше писать маленькие wrapper-функции:

```text
main -> твой wrapper -> libc function
```

Внутри wrapper проще контролировать:

- порядок аргументов;
- сколько байт положили;
- сколько байт выровняли;
- что `esp` восстановлен перед `ret`.

## Чеклист перед libc call

- [ ] аргументы положены справа налево;
- [ ] форматная строка передана адресом;
- [ ] после вызова стек очищен;
- [ ] `ebx/esi/edi` сохранены, если используются;
- [ ] перед `call` стек выровнен по 16 байт;
- [ ] после wrapper `esp` вернулся в исходное состояние.

## Типовые ошибки

| Ошибка | Симптом |
|---|---|
| стек не выровнен | локально может работать, в ejudge падать |
| забыли убрать padding | `ret` прыгает не туда |
| перепутали адрес и значение | `scanf/fscanf` пишет в мусор |
| забыли, что libc портит `eax/ecx/edx` | результат исчез после вызова |
