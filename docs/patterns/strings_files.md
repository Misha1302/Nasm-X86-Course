# Паттерн: строки и файлы через libc

## Когда нужен

- входной файл `data.in`, `input.txt`;
- выходной файл `output.txt`;
- строки, подстроки, разные строки;
- запрет `io.inc`.

Перед этим прочитай: [libc и alignment](./libc_alignment).

## Минимальные функции

| Функция | Зачем |
|---|---|
| `fopen` | открыть файл |
| `fclose` | закрыть файл |
| `fscanf` | читать из файла |
| `fprintf` | писать в файл |
| `strcmp` | сравнить строки |
| `strstr` | найти подстроку |
| `strlen` | длина строки |

## Буфер строки

```asm
section .bss
    s1 resb 1005
    s2 resb 1005
```

Для слова до 1000 символов нужен запас под `0` в конце.

## Массив коротких строк

Если строк не больше 500, длина не больше 10:

```asm
section .bss
    strings resb 500 * 16
```

Тогда адрес строки `i`:

```text
strings + i * 16
```

16 байт — удобно для адресации и даёт запас под `0`.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| `resb 1000` для строки длины 1000 | нет места под нулевой байт |
| сравнивать адреса вместо `strcmp` | разные буферы могут содержать одинаковый текст |
| забыть `fclose` | плохая привычка, особенно с output file |
| забыть alignment перед libc | Spring-04 может не принять |
