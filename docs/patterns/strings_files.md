# Паттерн: строки и файлы через libc

## Когда нужен

Когда в задаче есть:

- строки;
- подстроки;
- разные строки;
- `data.in`, `input.txt`, `output.txt`;
- запрет `io.inc`.

Перед этим прочитай: [libc и alignment](./libc_alignment).

## Нулевая строка

C-строка заканчивается байтом `0`.

```text
'a' 'b' 'c' 0
```

Поэтому для строки длины до 1000 нужен буфер минимум 1001 байт:

```asm
section .bss
    s1 resb 1005
    s2 resb 1005
```

Запас в несколько байт — нормальная учебная практика.

## Минимальные функции

| Функция | Зачем |
|---|---|
| `scanf("%s", s)` | прочитать слово из stdin |
| `fopen` | открыть файл |
| `fclose` | закрыть файл |
| `fscanf` | читать из файла |
| `fprintf` | писать в файл |
| `strcmp` | сравнить строки |
| `strstr` | найти подстроку |
| `strlen` | длина строки |

## Адрес строки

Если объявлено:

```asm
section .bss
    s1 resb 1005
```

то `s1` — это адрес первого байта. Для `scanf("%s", s1)` нужно передавать именно адрес:

```asm
push s1
push fmtString
call scanf
```

Не надо писать `push [s1]`: это прочитает первые 4 байта строки как число.

## Проверка подстроки

C-shape:

```c
if (strstr(a, b) != NULL)
    b is substring of a;
```

NASM-мысль:

```text
push needle
push haystack
call strstr
; eax = pointer or 0
```

Если `eax != 0`, подстрока найдена.

## Подсчёт разных строк

Если строк не больше 500, длина не больше 10, можно хранить фиксированные слоты:

```asm
section .bss
    strings resb 500 * 16
```

Почему 16:

```text
10 символов + 0 terminator + запас
адрес strings[i] = strings + i * 16
```

Алгоритм:

```text
unique_count = 0
for each input string s:
    found = false
    for i in 0..unique_count-1:
        if strcmp(s, strings[i]) == 0:
            found = true
    if not found:
        copy s to strings[unique_count]
        unique_count++
```

Для копирования можно использовать `strcpy` или ручной цикл по байтам.

## Файловый ввод

C-shape:

```c
FILE* f = fopen("data.in", "r");
while (fscanf(f, "%u", &x) == 1) {
    count++;
}
fclose(f);
printf("%u\n", count);
```

Главная проверка: `fscanf` возвращает количество успешно прочитанных элементов. Если вернул не `1`, чисел больше нет.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| `resb 1000` для строки длины 1000 | нет места под `0` |
| `push [s]` вместо `push s` | передал первые байты строки, а не адрес |
| сравнивать адреса вместо `strcmp` | одинаковые строки в разных буферах имеют разные адреса |
| забыть `fclose` | плохая привычка, особенно с output file |
| забыть alignment перед libc | Spring-04 может не принять |
| не проверять return value `fscanf` | цикл может стать бесконечным или считать мусор |

## Мини-практика

Что передать в `strstr`, чтобы проверить “является ли `b` подстрокой `a`”?

```text
haystack = a
needle   = b
```

То есть:

```asm
push b
push a
call strstr
```
