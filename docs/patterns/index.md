# Экзаменационные паттерны

Паттерны — это короткие мосты между базовым курсом и реальными задачами.

Базовые дни учат инструкциям и модели машины. Паттерны отвечают на вопрос:

```text
я уже знаю NASM, но как из этого собрать решение задачи?
```

## Как читать паттерн

Не надо заучивать файл целиком. Иди по порядку:

1. найди в условии признак паттерна;
2. пойми C++-shape;
3. переведи shape в NASM-шаблон;
4. проверь типовые ошибки;
5. только потом пиши полное решение.

## Карта паттернов

| Если в задаче | Открыть |
|---|---|
| запрещены условные переходы | [Branchless-маски](./branchless) |
| `K` битов, popcount, значащие нули | [Битовые циклы](./bit_counting) |
| палиндромы, дроби, GCD | [Десятичные алгоритмы](./decimal) |
| рекурсия без массивов | [Рекурсия](./recursion) |
| строки, файлы, libc | [Строки и файлы](./strings_files) |
| условие требует 16-byte stack alignment | [libc и alignment](./libc_alignment) |
| перенос сегмента, linked list через массивы | [Массивная связность](./array_linked_list) |
| function pointer, varargs | [Advanced stack](./advanced_stack) |
| результат больше 64 бит | [Big integer](./bigint) |

## Быстрая диагностика задачи

| В условии написано | Что это обычно значит |
|---|---|
| “запрещены инструкции условного управления” | нельзя `jcc`; нужен branchless-подход |
| `N <= 100000` | часто можно простой цикл |
| `N <= 1000000000` | часто нужен подсчёт по битам/комбинаторика |
| “массивы запрещаются” | возможно, нужна рекурсия или потоковая обработка |
| `input.txt`, `output.txt` | нужен `fopen/fscanf/fprintf/fclose` |
| “стек должен быть выровнен” | обычного CDECL-шаблона мало; смотри alignment |
| `void (*fn)(...)`, `...` | function pointer + varargs |
| ответ длиннее 10 цифр | проверь, помещается ли в 64 бита |

## Правило безопасности

Паттерн не заменяет решение. Он даёт форму мышления.

Плохо:

```text
нашёл похожий шаблон и вставил вслепую
```

Хорошо:

```text
понял, какие значения лежат в регистрах, где стек, где массив, где крайний случай
```
