# Паттерн: рекурсия в NASM

## Когда нужен

Когда условие говорит:

- “использовать массивы запрещается”;
- “нужно написать рекурсивную функцию”;
- надо вывести часть данных в обратном порядке, не сохраняя массив.

Рекурсия использует стек вызовов как память.

## Главная идея

Рекурсивная функция — это обычная CDECL-функция, которая вызывает сама себя.

```text
func -> func -> func -> base case
```

Каждый вызов получает свой frame:

```text
аргументы
return address
old ebp
локальные переменные
```

Поэтому значение, сохранённое в `[ebp-4]` в одном вызове, не конфликтует с `[ebp-4]` в другом вызове.

## Карта фрейма

```text
[ebp+12]  argument 2
[ebp+8]   argument 1
[ebp+4]   return address
[ebp]     old ebp
[ebp-4]   local saved value
```

Первый аргумент CDECL-функции — `[ebp+8]`, не `[ebp+4]`.

## Базовый шаблон

```asm
func:
    push ebp
    mov ebp, esp
    sub esp, 4          ; one local variable

    ; base case
    ; if (...) goto .base

    ; save value needed after recursion
    mov eax, [ebp+8]
    mov [ebp-4], eax

    ; recursive call
    push dword ...
    call func
    add esp, 4

    ; use saved value after recursive call
    mov eax, [ebp-4]

.base:
    mov esp, ebp
    pop ebp
    ret
```

## Печать до или после рекурсии

Это главный трюк.

```text
print before recursive call  -> прямой порядок
print after recursive call   -> обратный порядок
```

Пример:

```text
input: 1 2 3 0
```

Если печатать до вызова:

```text
1 2 3
```

Если печатать после вызова:

```text
3 2 1
```

## Как думать про задачу “разворот половины”

Идея обычно такая:

1. читаем очередное число;
2. если это `0` — базовый случай;
3. если позиция нечётная — печатаем сразу;
4. если позиция чётная — сохраняем число во фрейме и печатаем после рекурсии;
5. рекурсивный вызов читает следующую позицию.

Так массив не нужен: чётные элементы лежат в stack frames.

## Что обязательно сохранять

После `call` нельзя доверять:

```text
eax, ecx, edx
```

Если значение нужно после рекурсии, сохрани его:

```asm
mov [ebp-4], eax
```

или в callee-saved регистр, но тогда обязательно восстанови этот регистр.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| нет базового случая | бесконечная рекурсия и stack overflow |
| значение осталось только в `eax` | recursive call может его испортить |
| забыть `add esp, ...` после call | стек постепенно уедет |
| использовать `ebx/esi/edi` без сохранения | нарушение CDECL |
| читать следующий элемент после рекурсии | порядок ввода уже потерян |

