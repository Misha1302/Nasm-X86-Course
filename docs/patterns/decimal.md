# Паттерн: десятичные алгоритмы

## Когда нужен

Когда задача работает не просто с числом, а с его десятичной записью:

- развернуть число;
- проверить палиндром;
- получить цифры;
- сложить дроби;
- сократить дробь через GCD.

Главная инструкция здесь — `div` на 10.

## Как достать последнюю цифру

Для unsigned числа:

```text
digit = x % 10
x = x / 10
```

NASM:

```asm
xor edx, edx
mov ecx, 10
div ecx          ; eax = x / 10, edx = x % 10
```

Важно: `div` делит `edx:eax`, поэтому перед делением надо обнулить `edx`.

## Разворот числа

C++-shape:

```cpp
rev = 0;
while (x != 0) {
    digit = x % 10;
    rev = rev * 10 + digit;
    x /= 10;
}
```

NASM-shape:

```asm
xor esi, esi        ; rev = 0
mov ecx, 10

.loop:
    test eax, eax
    je .end

    xor edx, edx
    div ecx         ; eax = x / 10, edx = x % 10

    imul esi, 10
    add esi, edx
    jmp .loop

.end:
; esi = reversed number
```

Проверка:

```text
x = 123
rev = 0
step 1: digit=3, rev=3,   x=12
step 2: digit=2, rev=32,  x=1
step 3: digit=1, rev=321, x=0
```

## Палиндром

Число — палиндром, если оно равно своему развороту:

```text
is_palindrome(x) = reverse_decimal(x) == x
```

В задаче про гипотезу палиндромов важно слово “ровно”: если после `N` шагов палиндром — `Yes`, иначе `No`. Не надо останавливаться раньше и печатать `Yes`, если условие требует проверить именно результат после `N` шагов.

## GCD через Евклида

C++-shape:

```cpp
while (b != 0) {
    r = a % b;
    a = b;
    b = r;
}
```

NASM-shape:

```asm
; eax = a
; ebx = b
.loop:
    test ebx, ebx
    je .end

    xor edx, edx
    div ebx          ; edx = a % b

    mov eax, ebx     ; a = b
    mov ebx, edx     ; b = r
    jmp .loop

.end:
; eax = gcd
```

Если используешь `ebx` внутри CDECL-функции, сохрани его: `push ebx` / `pop ebx`.

## Сложение дробей

Если текущая сумма:

```text
num / den
```

и новая дробь:

```text
x / y
```

то:

```text
num = num * y + x * den
den = den * y
```

После каждого шага или в конце сокращаем:

```text
g = gcd(num, den)
num /= g
den /= g
```

Для задачи с маленьким `N` удобно сокращать после каждого добавления: числа меньше растут.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| забыть `xor edx, edx` перед `div 10` | делится `edx:eax`, результат будет мусорный или exception |
| потерять исходное число при palindrome check | надо сравнить reverse с исходником |
| не сохранить `ecx = 10` | если `ecx` используется ещё где-то, следующий `div` сломается |
| не сократить дробь | ответ не в требуемом виде |
| использовать signed `idiv` без причины | десятичные цифры для unsigned проще через `div` |

## Мини-практика

Разверни руками `120`:

```text
120 -> digit 0 -> rev 0
12  -> digit 2 -> rev 2
1   -> digit 1 -> rev 21
```

Почему результат `21`, а не `021`? Потому что число не хранит ведущие нули.
