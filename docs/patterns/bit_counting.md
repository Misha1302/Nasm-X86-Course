# Паттерн: битовые циклы

## Когда нужен

- найти лучший блок из `K` битов;
- посчитать единицы или нули;
- работать с двоичной записью числа;
- отличить brute force от комбинаторики.

## Sliding K-bit window

C++-shape:

```cpp
mask = (1u << K) - 1;
answer = 0;

for (shift = 0; shift <= 32 - K; ++shift) {
    candidate = (N >> shift) & mask;
    answer = max(answer, candidate);
}
```

NASM-мысль:

```text
candidate = N
candidate >>= cl
candidate &= mask
unsigned max через cmp/jae
```

## Popcount

```text
count = 0
while x != 0:
    count += x & 1
    x >>= 1
```

## Длина двоичной записи

```text
len = 0
while x != 0:
    len++
    x >>= 1
```

## Значащие нули

```text
zeros = bit_length(x) - popcount(x)
```

Для `x = 0` значащих битов обычно нет; в задачах диапазон часто начинается с `1`.

## Где граница

| Ограничение | Подход |
|---|---|
| `N <= 100000` | можно перебрать числа |
| `N <= 1000000000` | нужен подсчёт по битам и `C(n,k)` |

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| сдвигать на значение не в `cl` | x86 ждёт счётчик сдвига в `cl` |
| сделать `(1 << 32)` | недопустимо для 32-bit маски |
| использовать signed-сравнение для битового окна | нужно unsigned |
| считать ведущие нули значащими | значащие нули идут после первой единицы |
