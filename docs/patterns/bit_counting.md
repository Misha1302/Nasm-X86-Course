# Паттерн: битовые циклы

## Когда нужен

Когда задача говорит про:

- `K` подряд идущих битов;
- количество единиц;
- количество значащих нулей;
- двоичную запись числа;
- числа до `100000` или до `1000000000`.

Главный навык: превратить “двоичную запись” в цикл по битам.

## Бит как маленькое окно

Чтобы посмотреть младший бит:

```text
bit = x & 1
```

Чтобы перейти к следующему биту:

```text
x >>= 1
```

NASM-shape:

```asm
mov edx, eax
and edx, 1       ; edx = младший бит
shr eax, 1       ; следующий бит станет младшим
```

Для unsigned-чисел используй `shr`, не `sar`.

## Popcount: количество единиц

C++-shape:

```cpp
count = 0;
while (x != 0) {
    count += x & 1;
    x >>= 1;
}
```

NASM-shape:

```asm
xor ecx, ecx        ; count = 0

.loop:
    test eax, eax
    je .end

    mov edx, eax
    and edx, 1
    add ecx, edx

    shr eax, 1
    jmp .loop

.end:
; ecx = popcount
```

## Длина двоичной записи

Длина — это сколько раз можно сдвинуть число вправо до нуля.

```text
x = 1      -> len = 1    binary: 1
x = 4      -> len = 3    binary: 100
x = 18     -> len = 5    binary: 10010
```

C++-shape:

```cpp
len = 0;
while (x != 0) {
    ++len;
    x >>= 1;
}
```

Для `x = 0` обычно отдельно договариваются. В задачах “от 1 до N” ноль не входит.

## Значащие нули

Значащие нули — это нули **после первой единицы**.

```text
18 = 10010b
len = 5
ones = 2
significant zeros = 5 - 2 = 3
```

Формула:

```text
zeros = bit_length(x) - popcount(x)
```

## Sliding K-bit window

Задача: выбрать `K` подряд идущих битов так, чтобы полученное число было максимальным.

C++-shape:

```cpp
mask = (1u << K) - 1;
answer = 0;

for (shift = 0; shift <= 32 - K; ++shift) {
    candidate = (N >> shift) & mask;
    answer = max(answer, candidate);
}
```

Идея:

```text
N >> shift       ставит нужное окно в младшие биты
& mask           обрезает всё лишнее
unsigned max     выбирает лучший результат
```

NASM-важный момент: переменный сдвиг берёт счётчик из `cl`.

```asm
mov eax, [N]
shr eax, cl
and eax, [mask]
```

## Как сделать маску `(1 << K) - 1`

Если `1 <= K <= 31`:

```asm
mov eax, 1
mov ecx, [K]
shl eax, cl
sub eax, 1
; eax = mask
```

Не делай `1 << 32`: для 32-bit это опасная граница.

## Brute force или комбинаторика

| Ограничение | Подход |
|---|---|
| `N <= 100000` | можно перебрать `x = 1..N` |
| `N <= 1000000000` | полный перебор нельзя; нужен подсчёт по битам |

Для большой версии “двоичных нулей” идея такая:

```text
идём по битам N сверху вниз;
если видим 1, считаем сколько меньших чисел можно построить с нужным количеством нулей;
используем C(len, k)
```

Это уже алгоритмика, а не новая инструкция NASM.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| сдвигать на значение не в `cl` | x86 для variable shift ждёт `cl` |
| считать ведущие нули значащими | значащие нули начинаются после первой единицы |
| использовать signed max | битовые окна сравниваются как unsigned |
| забыть случай `K = 31` | маска большая, но всё ещё допустимая |
| brute force для `N = 1e9` | не пройдёт по времени |

## Мини-практика

Для `N = 12 = 1100b`, `K = 3`:

```text
shift 0: 100b = 4
shift 1: 110b = 6
shift 2: 011b = 3
answer = 6
```

Проверь руками, почему пример даёт `6`.
