# Advanced: многоразрядное произведение

::: warning
Это не базовый NASM-паттерн. Это отдельная advanced-тема.
:::

## Когда нужен

Когда результат не помещается в 32 бита и может не помещаться даже в `edx:eax`.

Например, произведение трёх signed 32-bit чисел может требовать до 96 бит.

## Идея

Храним число несколькими словами:

```text
value = high : middle : low
```

Минимальная стратегия:

1. определить знак результата;
2. взять абсолютные значения множителей;
3. перемножить с переносами;
4. если знак отрицательный — вывести `-`;
5. напечатать decimal через повторное деление большого числа на 10.

## Что надо уметь

- `imul` с расширенным результатом;
- `adc` для переноса;
- multiword shift/divide;
- decimal output вручную.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| оставить только `eax` | ответ обрежется до 32 бит |
| считать, что `edx:eax` всегда достаточно | для 3 множителей может не хватить |
| печатать binary value как `%d` | libc не знает твой 96-bit формат |
