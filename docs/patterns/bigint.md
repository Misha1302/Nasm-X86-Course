# Advanced: многоразрядное произведение

::: warning
Это не базовый NASM-паттерн. Это отдельная advanced-тема.
:::

## Когда нужен

Когда результат не помещается в 32 бита и может не помещаться даже в `edx:eax`.

Произведение трёх signed 32-bit чисел может требовать до 96 бит:

```text
32-bit * 32-bit * 32-bit -> до 96 бит
```

Значит нельзя просто сделать:

```asm
imul ebx
imul ecx
```

и напечатать `eax`.

## Главная идея

Храним большое число несколькими 32-bit словами:

```text
value = high : mid : low
```

Где:

```text
low  = младшие 32 бита
mid  = следующие 32 бита
high = старшие 32 бита
```

Это как число в системе счисления с основанием `2^32`.

## План решения signed product

1. Определить знак результата.
2. Взять абсолютные значения множителей.
3. Перемножить unsigned-значения с переносами.
4. Если знак отрицательный — вывести `-`.
5. Напечатать абсолютное значение в decimal.

## Знак результата

Знак отрицательный, если отрицательных множителей нечётное число.

Shape:

```text
sign = sign(a) xor sign(b) xor sign(c)
```

Для каждого множителя можно получить sign bit через `sar x, 31`, но аккуратно: само число тоже надо перевести в absolute value.

## Умножение с расширенным результатом

Для unsigned 32x32:

```asm
mov eax, a
mul dword [b]
; edx:eax = a * b
```

После первого умножения есть 64-bit результат:

```text
p = p_hi:p_lo
```

Потом нужно умножить 64-bit на 32-bit и получить до 96 бит.

Shape:

```text
p_lo * c -> contributes to low/mid
p_hi * c -> contributes to mid/high
add with carry
```

То есть нужны `add` и `adc`.

## Decimal output большого числа

`printf("%lld")` не поможет, если число больше 64 бит или лежит в твоём формате `high:mid:low`.

Нужен ручной вывод:

```text
while value != 0:
    digit = value % 10
    value = value / 10
    push digit
print digits in reverse order
```

Но деление теперь тоже многоразрядное: делим `high:mid:low` на 10 с переносом остатка сверху вниз.

Shape деления на 10:

```text
rem = 0
for word in [high, mid, low]:
    current = rem:word
    quotient_word = current / 10
    rem = current % 10
```

На x86 это удобно делать через `div`, где `edx:eax` — 64-bit делимое.

## Почему это advanced

Здесь сложность не в одной инструкции, а в дисциплине:

- не потерять перенос;
- не перепутать порядок слов;
- правильно обработать знак;
- правильно вывести decimal;
- отдельно обработать ноль.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| оставить только `eax` | ответ обрежется до 32 бит |
| считать, что `edx:eax` всегда достаточно | для 3 множителей может не хватить |
| забыть `adc` | перенос потеряется |
| печатать `%d` или `%lld` | libc не знает твой 96-bit формат |
| не обработать отрицательный знак отдельно | decimal output станет неправильным |
| не обработать `0` | цикл печати может ничего не вывести |

## Мини-практика

Почему `-234234 * -2394729 * -12323` отрицательное?

```text
минусов 3 -> нечётное количество -> результат отрицательный
```

Почему нельзя печатать только `edx:eax`? Потому что после трёх 32-bit множителей результат может занимать больше 64 бит.
