# Паттерн: branchless-маски

## Когда нужен

Когда в условии написано:

```text
Запрещается использовать инструкции условной передачи данных и управления.
```

Обычно это значит:

- нельзя `je`, `jne`, `jl`, `jg`, `jmp` для выбора ответа;
- нельзя `cmov`;
- можно использовать арифметику, биты, сдвиги, `and/or/xor/not`;
- можно заранее вывести формулу, которая работает без ветвления.

## Главная идея

Обычный `if` выбирает одно из двух значений:

```cpp
answer = condition ? a : b;
```

В branchless-решении вместо `condition` делают маску:

```text
mask = FFFFFFFFh, если надо выбрать a
mask = 00000000h, если надо выбрать b
```

Тогда выбор пишется так:

```text
answer = (a & mask) | (b & ~mask)
```

Почему работает:

| mask | `a & mask` | `b & ~mask` | answer |
|---|---|---|---|
| `FFFFFFFFh` | `a` | `0` | `a` |
| `00000000h` | `0` | `b` | `b` |

Важно: маска должна быть **0 или -1**, то есть `00000000h` или `FFFFFFFFh`. Маска `0/1` выбирает только младший бит.

## Базовый NASM-шаблон

```asm
; eax = a
; edx = b
; ecx = mask: 00000000h или FFFFFFFFh

and eax, ecx        ; eax = a & mask
not ecx             ; ecx = ~mask
and edx, ecx        ; edx = b & ~mask
or eax, edx         ; eax = answer
```

Если маска ещё нужна дальше, сначала сделай копию, потому что `not ecx` её портит.

## Как получить маску знака

Для signed 32-bit числа:

```asm
mov edx, eax
sar edx, 31
```

После этого:

```text
если eax >= 0: edx = 00000000h
если eax <  0: edx = FFFFFFFFh
```

`sar` важен: он размножает знаковый бит. `shr` здесь почти всегда ошибка.

## Пример: модуль без переходов

C++-идея:

```cpp
if (x < 0)
    answer = -x;
else
    answer = x;
```

Branchless-форма:

```text
mask = x >> 31
answer = (x ^ mask) - mask
```

NASM:

```asm
mov eax, [x]
mov edx, eax
sar edx, 31        ; edx = 0 или -1
xor eax, edx
sub eax, edx
```

Проверка на числах:

| x | mask | `x ^ mask` | `- mask` effect | answer |
|---:|---:|---:|---:|---:|
| `123` | `0` | `123` | `-0` | `123` |
| `-123` | `-1` | `~x` | `+1` | `123` |

## Пример: masked merge

Условие по битам:

```text
d[i] = c[i] ? a[i] : b[i]
```

Здесь `c` уже является маской по битам. Формула:

```text
d = (a & c) | (b & ~c)
```

NASM-shape:

```asm
mov eax, [a]
and eax, [c]

mov edx, [c]
not edx
and edx, [b]

or eax, edx
```

## Ceil division без ветвления

Для положительных чисел:

```text
ceil(a / b) = (a + b - 1) / b
```

Пример “сколько коробок нужно”:

```text
boxes = (items + D - 1) / D
```

NASM-shape:

```asm
mov eax, [items]
add eax, [D]
dec eax
xor edx, edx
div dword [D]
; eax = boxes
```

## Как решать branchless-задачу

1. Сначала напиши обычный C++ `if`.
2. Найди, что именно выбирается: число, количество, строка, маска.
3. Попробуй заменить выбор формулой.
4. Если нужен выбор между двумя числами — используй `(a & mask) | (b & ~mask)`.
5. Проверь маску на двух случаях: `0` и `FFFFFFFFh`.
6. Проверь, что в коде нет `jcc`, `cmov` и скрытого `if`.

## Частые ошибки

| Ошибка | Почему плохо |
|---|---|
| маска `0/1` | выбирается только один бит, а не всё число |
| `shr` вместо `sar` | для отрицательного числа получится `1`, а не `FFFFFFFFh` |
| `not mask`, а потом снова использовать mask | маска уже испорчена |
| вставить `cmp/jcc` “для удобства” | условие задачи нарушено |
| забыть unsigned/signed смысл | битовая формула может быть верной, а сравнение — нет |

## Мини-практика

Заполни пропуски:

```asm
; eax = x
mov edx, eax
___ edx, 31
xor eax, edx
sub eax, edx
```

Найди ошибку:

```asm
mov edx, eax
shr edx, 31
xor eax, edx
sub eax, edx
```
